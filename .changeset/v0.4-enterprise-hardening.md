# v0.4 — Enterprise hardening

20 fixes from the v0.3.1 code review.

## Breaking

- ButtonGroup role: `group` → `toolbar` (set via ElementInternals; query by tag,
  not `[role]`).
- Focus ring: `box-shadow` → `outline` (apps disabling the ring must use
  `outline: none`).
- Disabled `<tulpar-button href="…" disabled>` omits href attribute (was
  `javascript:void(0)`).
- Form submit: exactly one submit per click; `name`/`value` reflect into
  FormData like native `<button>`.
- Angular wrapper host: `display: contents` (layout transparent).
- Peer deps: Angular ^22, Vue ^3.5, lucide-angular >=0.500, no zone.js.
- Safari baseline: 17+.
- Keyframe rename: `tulpar-spin` → `tulpar-button-spin`.

## Fixes

- Form double-submit (#1).
- Vue auto-icon-only reactivity with whitespace-only / Comment vnodes (#2).
- `javascript:void(0)` removed from disabled anchor (#3).
- ButtonGroup `:scope > tulpar-button` selector (no nested capture) (#4).
- Focus delegation via `delegatesFocus` (#5).
- ButtonGroup MutationObserver-driven child sync (#6).
- `customElements.define` guard (micro-frontend safe) (#7).
- Hardcoded color fallbacks replaced with `@tulpar-ui/tokens/fallbacks` (#8).
- Outline focus ring + `forced-colors` support (#9).
- `console.warn` calls routed through dev-only helper (#11).
- Native `name` / `value` form parity (#12).
- Type unions deduplicated — wrappers import from `@tulpar-ui/core` (#13, #14).
- `ng-package.json` `$schema` dropped (#15).
- Angular wrapper has real TestBed-backed tests on @analogjs/vitest-angular (#17).
- Vue test uses a valid variant value (#18).
- Click listener attached in `connectedCallback`, removed in disconnect (#19).
- Keyframe renamed `tulpar-spin` → `tulpar-button-spin` (#20).
- Loading label span gets `aria-live="polite"` (#30).

## Features

- Angular wrapper rewritten signal-native: `afterRenderEffect()` + `viewChild()`
  for `autoIconOnly` (replaces `ngAfterContentInit`).
- Vue wrapper uses Vue 3.5 reactive props destructure.
- New `name` and `value` properties on `<tulpar-button>` matching native button
  semantics.

## Docs

- Tooltip limitations documented on `tooltip` prop JSDoc and in README (#10).
- README: browser baseline (Safari 17+), wrapper transparency section.
- Migration guide: `docs/migration/v0.3-to-v0.4.md`.
