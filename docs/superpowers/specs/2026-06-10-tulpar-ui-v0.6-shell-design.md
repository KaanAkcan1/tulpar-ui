# Tulpar UI v0.6 — Application Shell (`@tulpar-ui/shell`) Design Spec

**Date:** 2026-06-10
**Status:** Approved (brainstorm complete, ready for implementation plan)
**Scope:** v0.6 — Shell ailesi (`tulpar-shell`, `tulpar-topbar`, `tulpar-sidenav`, `tulpar-nav-item`) + altyapı/kalite işleri (Bölüm B)

---

## 1. Overview & Goals

### Problem

Kaan'ın projeleri (ör. connectIk) her seferinde aynı app-layout kodunu (topbar + sidebar + menü + responsive davranış) kopyalayarak başlıyor. Kopyalar zamanla ayrışıyor; bir projedeki düzeltme diğerine taşınmıyor. PrimeTek bu katmanı paket olarak satmıyor (Sakai bir şablon repo) — pazarda da gerçek bir boşluk.

**Hedef:** App shell'i Tulpar UI ürün paketi olarak bir kez yazmak; Vue'da `App.vue`'da, Angular'da `app.component`'te birkaç satır deklarasyonla tüketmek.

### Önemli çerçeve notları

- **connectIk yalnız yetenek referansıdır.** "Neler yapabilmeli" listesini oradan aldık (menü filtreleme, dark toggle, overlay davranışı, configurator deneyimi). **Görsel/tema referansı DEĞİLDİR** — görsel tasarım sıfırdan, enterprise seviyede, Tulpar'ın kendi kimliğiyle yapılır.
- **Tasarım kalite direktifi:** Plan ve implementasyon aşamalarında görsel/UX kararı gereken her noktada `/frontend-design` ve `/ui-ux-pro-max` skill'leri kullanılır. Hedef çıta: product-ready, enterprise seviye.
- **Doğrulama:** Implementasyon bittikten sonra playground-ng ve playground-vue tam shell demo sayfalarıyla güncellenir; gerçek kullanım orada ispatlanır.

### Out of scope (v0.6'da yok)

- Configurator paneli (v2 — Select/Switch/Checkbox bileşenleri gerektirir, onlar v0.7+)
- Runtime renk preset API'si (PrimeVue `definePreset` benzeri — tokens paketinin ayrı spec'i)
- `top` / `mix` layout modları (API isimlendirmesi genişlemeye uygun seçilir, davranış yazılmaz)
- Breadcrumb, kullanıcı menüsü/dropdown, ayrı footer bileşeni (footer slot'u var)
- RTL
- Public Tooltip / Badge / Icon bileşenleri (shell içi ihtiyaçlar içeride çözülür)

## 2. Research Summary (10 enterprise ürün)

| Ürün | Model | Alınan ders |
|---|---|---|
| Vaadin AppLayout + SideNav | Saf WC, slot tabanlı | Referans mimari; aktif item URL'den; overlay kapatma event'i içeride |
| SAP UI5 (webcomponents-fiori) | WC, item'lar element | Shell ailesi **ayrı pakette** (temel bileşenlerden izole) |
| IBM Carbon UI Shell | WC | **Rail modu** (ikon şeridi + flyout); header'da ürün→global hiyerarşi |
| Ionic split-pane/menu | WC + resmi ng/vue wrapper | Breakpoint media-query string prop'u (`when`); Tulpar'la aynı dağıtım stratejisi |
| Ant Design ProLayout | Config-driven | `layout: side\|top\|mix` genişleme alanı; açık prop isimleri |
| MUI Toolpad DashboardLayout | Provider + adapter | **Router adapter deseni**; navigation/branding JSON |
| Mantine AppShell | Slot + breakpoint config | `collapsed: {desktop, mobile}` state modeli |
| Quasar QLayout | 11 karakterlik view string | **Anti-örnek:** kriptik config DX'i öldürür |
| PrimeVue (Sakai + v4 preset) | Şablon, paket değil | Boşluğun kanıtı; 3 katmanlı token mimarisi Tulpar'la örtüşüyor |
| Telerik Kendo Drawer | Data + template override | "JSON prop slot'u kaldırmaz, üstüne gelir" ilkesinin teyidi |

**Ortak desenler (kararlara temel):** (1) çift API: data-config + slot, (2) router adapter/köprü — layout router bilmez, (3) üç sidebar modu standart: static/overlay/rail + mobilde otomatik overlay, (4) configurator hiçbir üründe shell'e gömülü değil, (5) localStorage persistence hiçbir kütüphanede default değil.

**ui-ux-pro-max doğrulaması:** Kararlar tarandı; iki boşluk bulunup kapatıldı: **skip-link** (shell otomatik render eder) ve **z-index token ölçeği** (keyfi değer yasak). Diğer kurallar (active state, keyboard nav, adaptive navigation, reduced motion, scrim %40-60, 44px hedef) tasarımda karşılanıyor.

| # | Karar | Gerekçe |
|---|---|---|
| 1 | Tulpar UI **ürün paketi** — diğer bileşenlerle aynı çıta | Ticari hedef; "pragmatik iç paket" reddedildi |
| 2 | Ayrı **`@tulpar-ui/shell`** paketi; core'a bağımlı değil | UI5/Toolpad emsali; core yalın kalır; linked versioning maliyeti sıfırlar |
| 3 | v1 kapsamı **çekirdek dörtlü:** shell, topbar, sidenav, nav-item | Vaadin/Carbon ölçeği; footer slot olarak shell'de |
| 4 | Tema kapsamı **Toolpad modeli:** dark toggle + layout config + opt-in persistence | Runtime renk preset'i tokens paketinin ayrı işi |
| 5 | Menü **çift API:** `<tulpar-nav-item>` element'leri + `items` JSON prop'u | Kayıtlı ilke: prop slot'u kaldırmaz, üstüne gelir |
| 6 | Router köprüsü: **gerçek `<a href>` + cancelable `tulpar-navigate` event** + wrapper'larda otomatik router bağlama | SEO, orta-tık, back-button bedava; uygulama tarafında sıfır kod |
| 7 | Sidebar modları **üçü de v1'de:** static, overlay, rail; mobilde otomatik overlay | Rail'i sonradan eklemek CSS grid'ini kırar |
| 8 | State modeli Mantine tarzı: `{ sidenavMode, collapsed: {desktop, mobile}, dark }` | connectIk'in 5 dağınık boolean'ının temizlenmiş hali |
| 9 | Persistence **opt-in:** `persistShellState()` util / wrapper'da `persistKey` | 10 üründe de default değil; SSR/çok-uygulama sürprizi yok |
| 10 | Shell'e **`aside` slot'u** (sağ panel): `aside-open`, scrim, Escape, z-index shell'in işi | Configurator/bildirim/filtre panelleri için genel altyapı |
| 11 | İçerik prop'ları: `content-width="fluid\|fixed"`, `content-padding="none\|compact\|comfortable"` | ProLayout deseni; data-dense dashboard ihtiyacı |
| 12 | `footer-mode="inline\|fixed"` | Hem connectIk inline footer'ı hem sabit bottom bar |
| 13 | JSON item ikonları **`iconClass`** (ikon fontu sınıfı) + tam kontrol için slot | connectIk menü verisi (`pi pi-home`) olduğu gibi taşınır; Tulpar ikon sistemi kurmaz |
| 14 | Shell otomatik **skip-link** render eder | ui-ux-pro-max `skip-links` kuralı |
| 15 | **Z-index token ölçeği:** `--tulpar-shell-z-{topbar,sidenav,mask,aside}` | ui-ux-pro-max `z-index-management`; Shadow DOM stacking context notu |
| 16 | Shell'in iç ihtiyaçları içeride: hamburger butonu, badge span'i, rail flyout | Bağımlılıksız kurulabilir paket; Vaadin `drawer-toggle` emsali |
| 17 | Yetki/menü filtreleme **pakete girmez** — uygulama `items`'ı filtreleyip verir | connectIk `featureKey` deseni app concern; spec'te desen olarak dokümante edilir |

## 4. Architecture

```
┌──────────────────────────────────────────────────┐
│  @tulpar-ui/tokens                               │
│  + shell semantic token grubu (yeni)             │
└──────────────────────────────────────────────────┘
            ▲                        ▲
            │                        │
┌───────────────────────┐  ┌───────────────────────────┐
│ @tulpar-ui/core       │  │ @tulpar-ui/shell (YENİ)   │
│ button, inputs...     │  │ tulpar-shell              │
│ (shell'den bağımsız)  │  │ tulpar-topbar             │
└───────────────────────┘  │ tulpar-sidenav            │
            ▲              │ tulpar-nav-item           │
            │              │ persistShellState()       │
            │              └───────────────────────────┘
            │                        ▲
     ┌──────┴────────────────────────┴──────┐
     │ @tulpar-ui/angular   @tulpar-ui/vue  │
     │ (mevcut paketlere shell wrapper'ları │
     │  + router köprüleri eklenir)         │
     └──────────────────────────────────────┘
```

- `@tulpar-ui/shell` runtime'da yalnız Lit'e bağımlı; tokens build-time/peer.
- Layout **CSS Grid** ile kurulur (fixed+padding telafisi değil) — sticky nav'ın içeriği örtmesi yapısal olarak imkânsız.
- Rail flyout pozisyonlama mantığı ileride public Tooltip ile paylaşılabilir → `_internal` katmanlama kararı (Bölüm B) bunun için netleştirilir.

## 5. Component API'leri

### 5.1 `<tulpar-shell>`

**Slots:** `topbar` (herhangi bir markup — `tulpar-topbar` zorunlu değil), `sidenav`, default (içerik), `footer`, `aside`

**Props/Attributes:**

| Prop | Tip / Değerler | Default | Açıklama |
|---|---|---|---|
| `sidenav-mode` | `static \| overlay \| rail` | `static` | Mobil eşik altında her mod overlay'e düşer |
| `mobile-breakpoint` | media query string | `(max-width: 991px)` | Ionic `when` modeli |
| `sidenav-collapsed` | boolean | `false` | Desktop collapsed state (mobile state içeride) |
| `aside-open` | boolean | `false` | Sağ panel görünürlüğü |
| `content-width` | `fluid \| fixed` | `fluid` | `fixed`'te max-width token'dan |
| `content-padding` | `none \| compact \| comfortable` | `comfortable` | |
| `footer-mode` | `inline \| fixed` | `inline` | |
| `dark` | boolean | `false` | `document.documentElement`'te `.dark` yönetimi (View Transitions varsa onunla) |

**Events:** `tulpar-shell-change` (state değişimi, `detail: ShellState`), `tulpar-aside-close`

**Davranış sorumlulukları:** mobil overlay geçişi, scrim (%40-60 opaklık), dışarı-tıklama + Escape ile kapanma, overlay açıkken focus yönetimi, skip-link ("İçeriğe atla" → içerik alanı), z-index katmanları, `prefers-reduced-motion`, transform/opacity tabanlı animasyon (ease-out giriş / ease-in çıkış, 150-300ms).

### 5.2 `<tulpar-topbar>`

**Slots:** `start`, default (orta), `end`
**Props:** `show-menu-button` (hamburger; shell ile event üzerinden senkron, görünür focus ring'li, ≥44px hedef)
**Events:** `tulpar-menu-toggle`

### 5.3 `<tulpar-sidenav>`

**Slots:** `header`, default (item'lar), `footer` (alt-hizalı secondary section — UI5 deseni)
**Props:** `items?: TulparNavItemData[]` — verilirse item'ları kendisi render eder; slot'la birlikte de kullanılabilir
**Davranış:** klavye gezinmesi (ok tuşları + Tab), `<nav>` landmark + `aria-label`, rail modunda ikon-only + hover/focus flyout etiket

```ts
interface TulparNavItemData {
  label: string;
  href?: string;
  iconClass?: string;          // ör. "pi pi-home" — <i class> olarak render edilir
  badge?: string;
  items?: TulparNavItemData[]; // iç içe grup
  disabled?: boolean;
  target?: string;
  active?: boolean;            // manuel override; default URL eşleşmesi
}
```

### 5.4 `<tulpar-nav-item>`

**Props:** `href`, `label`, `icon-class`, `badge`, `active`, `disabled`, `target`
**Slots:** `icon` (tam kontrol), default (children olarak iç içe `tulpar-nav-item` → açılır-kapanır grup, `aria-expanded`)
**Render:** gerçek `<a href>`; grup başlıkları buton semantiğiyle
**Events:** `tulpar-navigate` — `composed, bubbles, cancelable`, `detail: { href, item }`. İptal edilmezse normal `<a>` navigasyonu çalışır.
**Aktif vurgu:** URL eşleşmesinden otomatik (`aria-current="page"`), renk **+** bar göstergesi (yalnız renk değil); item yüksekliği ≥44px.

### 5.5 `persistShellState(shellEl, key)`

Opt-in util: shell state'ini localStorage'a senkronlar, sayfa açılışında geri yükler. Çağrılmazsa hiçbir storage erişimi yapılmaz.

## 6. Davranış Sorumluluk Matrisi

| Davranış | Sahibi | Doğrulayan kural |
|---|---|---|
| Router entegrasyonu | Angular/Vue wrapper (`tulpar-navigate` → router) | `back-behavior`, `deep-linking` |
| Mobil overlay geçişi | `tulpar-shell` | `adaptive-navigation` |
| Scrim / dışarı-tıklama / Escape | `tulpar-shell` | `modal-escape`, `escape-routes` |
| Aktif menü vurgusu | `tulpar-nav-item` | `nav-state-active`, `aria-current` |
| Hamburger ↔ sidebar senkronu | `tulpar-topbar` ↔ shell event'i | `nav-hierarchy`, focus ring |
| Skip-link | `tulpar-shell` | `skip-links` |
| localStorage senkronu | `persistShellState()` (opt-in) | navigasyon durumu URL'de, tercih storage'da |

## 7. Wrappers

### Angular (`@tulpar-ui/angular`)

- `TulparShellNg`, `TulparTopbarNg`, `TulparSidenavNg`, `TulparNavItemNg` — selector'lar `tulpar-*-ng` konvansiyonunda
- **Yalnız signal API:** `input()`, `output()`, `model()` — RxJS yok (kayıtlı kural)
- `[(asideOpen)]` two-way binding (`model()`)
- `persistKey` input'u → içeride `persistShellState` çağrısı
- Router köprüsü: wrapper `tulpar-navigate`'i yakalar → `preventDefault()` + `Router.navigateByUrl`; `NavigationEnd`'de aktif item güncellenir

### Vue (`@tulpar-ui/vue`)

- `TulparShell`, `TulparTopbar`, `TulparSidenav`, `TulparNavItem` SFC'leri
- `v-model:aside-open`, `persist-key` prop'u
- Router köprüsü: `useRouter()` varsa `tulpar-navigate` → `router.push`; yoksa native navigasyona düşer (router'sız kullanım da çalışır)

## 8. Theming & Tokens

- Tokens paketine **shell semantic token grubu** eklenir: `--tulpar-shell-sidenav-width`, `--tulpar-shell-sidenav-rail-width`, `--tulpar-shell-topbar-height`, `--tulpar-shell-content-max-width`, `--tulpar-shell-z-{topbar,sidenav,mask,aside}`, sidenav/topbar yüzey-metin renkleri.
- Component CSS **yalnız semantic token** kullanır + `var()` fallback (mevcut kural).
- Dark mode mevcut `.dark` tetikleyicisiyle; brand `[data-brand]` ile — shell'e özel mekanizma yok.
- Görsel dil connectIk'ten **alınmaz**; `/frontend-design` + `/ui-ux-pro-max` ile plan aşamasında tasarlanır (enterprise, data-dense dashboard'a uygun, "ornate design" anti-pattern'inden uzak).

## 9. Accessibility Requirements

- Sidenav: `<nav>` + `aria-label`; gruplar `aria-expanded`; aktif item `aria-current="page"`
- Klavye: tam gezinme, tab sırası görsel sırayla aynı, klavye tuzağı yok
- Overlay/aside açıkken: focus panele taşınır, Escape kapatır, focus geri döner
- Skip-link ilk Tab'da görünür
- Rail: ikon-only item'larda erişilebilir etiket korunur (flyout + `aria-label`)
- Dokunma hedefleri ≥44px; scrim %40-60; `prefers-reduced-motion` desteği
- Kontrast: AA (4.5:1 metin, 3:1 büyük öğe) — her iki modda ayrı doğrulanır
