# Tulpar UI v0.13 — Display & Status Atoms

**Status:** Reviewed (spec-review loop passed, 3 iterations) — pending user approval
**Date:** 2026-06-22
**Predecessor:** v0.12 Toast + Message (transient feedback family)
**Type:** New atom family (core elements) + shared internal tone module + tokens + Angular/Vue wrappers
**Roadmap:** Wave 1 of the path to the AG-Grid-equivalent data grid (see [`docs/roadmap.md`](../../roadmap.md))

---

## 1. Overview

Seven small, high-frequency presentational atoms. They give every page enterprise completeness now,
and double as the **status / loading vocabulary of the future data grid** (cell renderers + loading
states) — which is why they come first in the roadmap.

| Atom | Element | Job | Interactive |
|---|---|---|---|
| **Tag** | `<tulpar-tag>` | Static tonal metadata label | No |
| **Badge** | `<tulpar-badge>` | Count / status indicator (number · dot · "99+") | No |
| **Chip** | `<tulpar-chip>` | Interactive label (clickable / removable) | Yes |
| **Avatar** | `<tulpar-avatar>` | Image / initials / icon identity | No |
| **Skeleton** | `<tulpar-skeleton>` | Loading placeholder | No |
| **Spinner** | `<tulpar-spinner>` | Indeterminate loader | No |
| **Progress** | `<tulpar-progress>` | Linear + circular progress (det. / indet.) | No |

### Goals
- Seven correct, accessible, on-brand atoms consistent with the shipped tone/size conventions.
- Hold up at **grid-cell density** (small sizes, hundreds of instances, truncation, no layout shift).
- A distinctive but enterprise-calm visual language (square-ish tags, serif-monogram avatars).
- WCAG 2.2 AA: every loading/status state has an accessible name; meaning never by color alone;
  reduced-motion and forced-colors handled per atom.

### Non-Goals (binding) — see §12
ChipGroup + `selectable` chips · AvatarGroup-as-element · Badge attached-to-host rendering · Tree ·
Timeline · Rating · any data grid. Some of these have **reserved API surface** (noted inline) so they
are additive later, not breaking.

### Cross-cutting principle: visual choices are props, default = recommended
Shape, variant, tone, and size are all **props** with the recommended option as the default. Consumers
change the look without forking styles. (Established preference: [[feedback_optional_prop_does_not_remove_slot]].)

## 2. Architecture

```
@tulpar-ui/core
  src/_internal/tone/            ← package-private tone resolver (promoted from toast), prefix-parameterized
  src/tag/tulpar-tag.ts(.styles.ts,.test.ts)
  src/badge/tulpar-badge.ts(…)
  src/chip/tulpar-chip.ts(…)
  src/avatar/tulpar-avatar.ts(…)
  src/skeleton/tulpar-skeleton.ts(…)
  src/spinner/tulpar-spinner.ts(…)
  src/progress/tulpar-progress.ts(…)
@tulpar-ui/tokens
  src/semantic/atom.ts           ← new semantic block: atom.tone.* (+ size scales)
  src/semantic/atom.contrast.test.ts
@tulpar-ui/angular  → <tulpar-tag-ng> … (signal wrappers, prop+slot forwarding)
@tulpar-ui/vue      → <TulparTag> … (SFC wrappers, slot carrier forwarding)
```

### 2.1 Shared tone module (`_internal/tone/`) — the one real architectural change

Today tonal logic lives in `toast/tone-resolver.ts` (~250 lines) hard-coded to emit `--tulpar-toast-*`.
Tag/Chip/Badge are the **second consumer** — the `_internal` rule's "promote on a real second consumer"
trigger ([`docs/architecture/internal-layering.md`](../../architecture/internal-layering.md)).

- **Promote** the resolver to `core/src/_internal/tone/` and **parameterize the CSS-var prefix**. This
  is a real signature change (not a string swap): the literal `--tulpar-toast-*` keys baked into
  `resolveTone`/`applyOverrides` become prefix-driven. New shape:

  ```ts
  resolveTone(input: ToneInput, opts: { prefix: string }): ToneResult
  //   prefix 'toast' → --tulpar-toast-surface-l …   (byte-identical to today)
  //   prefix 'tag'   → --tulpar-tag-surface-l …
  ```
  Toast is refactored to pass `prefix: 'toast'` → identical output, so its existing tests are the
  regression gate. **In-package** move — the `_internal` no-cross-package-import boundary holds.
- **`neutral` is a new built-in tone.** Today `BuiltinTone = info|success|warning|danger` (no neutral).
  The union gains `neutral`; because the built-in path is attribute-only (returns `{builtin:true, vars:{}}`),
  adding `neutral` is **type-only** in the resolver — the actual neutral rendering lives entirely in the
  atom token layer (§7, incl. the inverted-contrast solid treatment). Toast never uses `neutral`, so its
  behavior is unchanged.
- **Two consumer shapes** are supported explicitly:
  - **Quartet shape** (Tag/Chip/Badge): `{surface, text, border, solidBg, solidText}` with light/dark.
  - **Single-accent shape** (Avatar palette color, Spinner/Progress accent): one resolved color.
    Avatar's deterministic fallback color reuses the brand-family list (NOT the toast quartet path).
- **Built-in tones are attribute-only** (`[tone="success"]` → semantic tokens that auto-flip for `.dark`
  via the generated sheet). **No inline custom properties** are written for built-in tones — this is the
  hot path (hundreds of grid cells) and must stay zero-inline-style. `custom` tone (brand family / raw
  color) is the **escape hatch** that writes the `-l/-d` inline vars, documented as the expensive path.
- **No `:host-context(.dark)` on the high-volume atoms.** Dark mode is driven purely by semantic-token
  auto-flip (the token sheet already swaps under `.dark`). `:host-context` is reserved for the custom
  inline-var path only (rare), avoiding its known perf/portability cost at grid scale.

### 2.2 Unit boundaries
Each atom is an independent Lit element: props in → shadow render out, no shared runtime state, no cross
-atom imports. The only shared code is `_internal/tone/` (pure, DOM-free, unit-tested in isolation) and
the token sheet. Every atom is understandable and testable without reading another's internals.

## 3. Shared visual system

### 3.1 Tone vocabulary
`neutral | info | success | warning | danger | custom`. **Atoms default to `neutral`** (toast/overlay
keep their own defaults). `custom` accepts a brand-family name (`al·gok·ulgen·otuken·kizagan·erlik·ilay·umay…`)
or a raw CSS color, plus `bg`/`accent`/`text` part overrides (same contract as toast). Tone always pairs
with a dot/icon or text — **never conveyed by hue alone** (colorblind + forced-colors safety). *(The
family list here is the generic `custom`-tone example; Avatar's deterministic fallback uses its own
curated list — see §4.4.)* For `custom` tone, `color` (family name or CSS color) is the accent override
on Tag/Chip/Badge/Spinner/Progress; **Avatar exposes `color` as a first-class prop** (single-accent) —
the one exception.

### 3.2 Variants (Tag / Chip / Badge)

| Variant | Treatment (light) | Treatment (dark) | Default for |
|---|---|---|---|
| `soft-tonal` | bg `family[100]`, text `family[800]`, border transparent (opt. `[200]` hairline) | bg `family[900]`, text `family[100]`, hairline `family[700]` | **Tag, Chip** |
| `outline` | transparent bg, text `family[700]`, border `family[400]` (≥3:1) | text `family[300]`, border `family[500]` | opt-in |
| `solid` | bg = `atom.tone.{t}.solid.bg`, text = `…solid.text` token slots | bg `family[500]`, text `yagiz[950]` | **Badge** |
| `ghost` *(Chip only)* | transparent → `family[100]` on hover, text `family[700]` | transparent → `family[900]` on hover | toolbar/filter chips |

- **Allowed sets:** Tag = {soft-tonal, outline, solid}; **Chip** = those + `ghost`; **Badge** = {solid, soft-tonal, outline}.
- Soft uses the **`[100]` map** (overlay), not the `[50]` map (toast/feedback) — `[50]` disappears on
  zebra grid rows. Validated by the new contrast test (§7).
- **Solid bg/text are explicit token slots** (§7), not runtime step math. `solid.text` ink rule: white
  for info/success/danger; **dark ink (`yagiz[900]`)** for warning + brand-green (luminance).
- **Neutral solid** is the known landmine (reads "disabled," esp. dark): rendered as an **inverted
  `contrast` chip** (`colpan[50]`/`bg-surface` fill, `yagiz[900]`/text ink) rather than mid-gray.

### 3.3 Shape (prop; default = recommended)
- **Tag / Chip:** `shape="square"` (default — `radius.lg` 6px rounded-rect) | `"pill"` (fully rounded) | `"sharp"` (`radius.sm` ~2px crisp corners).
- **Badge:** pill by default (counts need the lozenge); `shape="square"` available for parity.
- **Avatar:** `shape="rounded-square"` (default — 4–10px by size) | `"circle"`.
- The deliberate **square-tag / pill-badge contrast** is an intentional identity signal.

### 3.4 Size scale (px) — `xs · sm · md · lg · xl`
Consistent with the selection family's 5-step scale. **`sm` is the grid-cell renderer; `md` is the
standalone default.** At `xs`, icon+label+remove crowd — `xs` is for dot/short-label or icon-only.

**Tag / Chip** — height / pad-x / font / icon / gap / radius(square):

| | xs | sm | md | lg | xl |
|---|---|---|---|---|---|
| height | 16 | 20 | 24 | 28 | 32 |
| pad-x | 5 | 6 | 8 | 10 | 12 |
| font | 11 | 12 | 13 | 14 | 15 |
| icon | 11 | 12 | 14 | 14 | 16 |
| gap | 3 | 4 | 6 | 6 | 8 |
| radius | 4 | 5 | 6 | 6 | 7 |

Weight `500`, `letter-spacing: 0`, **no uppercase/track-out** (Bootstrap-badge tell). md font 13px is intentional.

**Avatar** — size / initials font / radius(rounded-square):

| | xs | sm | md | lg | xl |
|---|---|---|---|---|---|
| size | 20 | 24 | 32 | 40 | 48 |
| initials | 9 | 10 | 13 | 16 | 19 |
| radius | 4 | 5 | 6 | 8 | 10 |

**Badge** — dot Ø: sm 6 / md 8 / lg 10. Count: height sm 16 / md 18 / lg 20, min-width = height (single-digit
lozenge), pad-x 5–6, font 11/11/12, `font-variant-numeric: tabular-nums`.

**Spinner** — Ø: xs 14 / sm 16 / md 20 / lg 24 / xl 28, stroke ≈ size×0.11 (round-cap).

**Progress** — linear thickness: thin 4 / regular 6 (default) / thick 8. Circular: sm 32 / md 44 / lg 56, stroke 4.

### 3.5 Motion (all with reduced-motion fallback; primitive easings)
- **Skeleton:** directional **100° shimmer sweep**, `1.4s linear infinite`. Reduced-motion → static 8%
  diagonal tint (no movement). Stagger only ≤6 lines (`index×90ms`), capped; grids animate in-sync.
- **Spinner:** quarter-arc (`stroke-dasharray` ~25/75), `rotate 0.7s linear infinite` (matches switch
  spinner). Faint track ring at `currentColor/0.12`. Reduced-motion → 3-dot opacity cycle (1.2s).
- **Progress:** determinate value transition `320ms cubic-bezier(.2,0,0,1)`; indeterminate linear = one
  40%-width traveling bar `1.8s`; indeterminate circular reuses the spinner arc. Reduced-motion →
  determinate snaps with 120ms fade; indeterminate → opacity pulse on a static track.

### 3.6 Dark mode & forced-colors
- Dark = semantic-token auto-flip (no `:host-context` on built-in tones). Dark soft keeps a `family[700]`
  hairline; dark solid drops to `[500]` + `yagiz[950]` ink.
- **No drop shadows** on Tag/Chip/Badge (inline-atom over-design tell). The only `box-shadow` is the
  **surface-colored knockout ring** on stacked avatars / corner badges (swaps with mode).
- `@media (forced-colors: active)` per atom: tonal surfaces collapse → explicit `1px solid CanvasText`
  border + `Canvas`/`CanvasText`; solid badge inverts (`CanvasText` bg / `Canvas` text); selected chip →
  `Highlight`; focus rings → `CanvasText`; skeleton → `1px dashed CanvasText` shape outline (gradient is
  invisible in HCM); spinner/progress stroke → `CanvasText`, track → `GrayText` (motion kept).
- Focus: interactive parts use `:focus-visible` + the existing `--tulpar-color-focus-ring` semantic
  token (defined in `semantic/color.ts`, used by Button) — 2px ring, 2px offset, HCM fallback `CanvasText`.

## 4. Per-atom API

> Conventions: every content prop has a matching slot (slot wins when both set). `tone`, `size`,
> `variant`, `shape` defaults are bold. Layout uses logical properties (RTL-ready, though RTL is out of scope).

### 4.1 `<tulpar-tag>` — static metadata (read-only)

| Prop | Type | Default | Notes |
|---|---|---|---|
| `tone` | tone vocab | **`neutral`** | + `color/bg/accent/text` for `custom` |
| `variant` | `soft-tonal\|outline\|solid` | **`soft-tonal`** | |
| `shape` | `square\|pill\|sharp` | **`square`** | `sharp` = ~2px crisp |
| `size` | `xs…xl` | **`md`** | |
| `dot` | boolean | `false` | leading tone dot |
| `icon` | string | — | name / raw SVG / emoji; mutually exclusive with `dot` (icon wins) |
| `disabled` | boolean | `false` | dimmed; for disabled toolbars |
| `label` | string | — | alias of default slot content |

- **Slots:** default (label) · `icon`. **No remove control** (Tag is fully static; removal is Chip's job).
- **Truncation:** `max-width` (CSS var `--tulpar-tag-max-width`) + ellipsis + `min-width:0`; when the
  label overflows, the element sets the **native `title` attribute** to the full text. (No dependency on
  the v0.11 Tooltip element — preserves the §2.2 no-cross-atom-import boundary.)
- **A11y:** non-interactive; renders as inline text. Color never the sole signal (pair `dot`/`icon`/text).

### 4.2 `<tulpar-badge>` — count / status

| Prop | Type | Default | Notes |
|---|---|---|---|
| `tone` | tone vocab | **`neutral`** | |
| `variant` | `solid\|soft-tonal\|outline` | **`solid`** | soft/outline for low-emphasis counts |
| `count` | number | — | numeric mode |
| `max` | number | `99` | `count > max` → `"{max}+"` |
| `show-zero` | boolean | `false` | when false, the badge is not rendered at `count = 0` (no exit animation) |
| `dot` | boolean | `false` | dot mode (ignores `count`) |
| `shape` | `pill\|square` | **`pill`** | |
| `size` | `sm\|md\|lg` | **`md`** | |
| `label` | string | — | short status text (alias of slot); constrained to short content |

- **Slots:** default (number/short label). **Placement API reserved:** a `placement` prop
  (`top-end`/`top-start`/…) and host-anchored rendering are **reserved names, not implemented** in v0.13
  (attached mode is Wave-2+); standalone badge ships now.
- **A11y:** count badge exposes an accessible name including the noun via `aria-label` (e.g.
  `"3 unread"`), reusing the toast count pattern; `tabular-nums` prevents width jitter.

### 4.3 `<tulpar-chip>` — interactive label

| Prop | Type | Default | Notes |
|---|---|---|---|
| `tone` | tone vocab | **`neutral`** | |
| `variant` | `soft-tonal\|outline\|solid\|ghost` | **`soft-tonal`** | `ghost` = transparent→hover (toolbars) |
| `shape` | `square\|pill\|sharp` | **`square`** | |
| `size` | `xs…xl` | **`md`** | |
| `icon` | string | — | leading icon (name/SVG/emoji) |
| `avatar` | string | — | leading avatar (image URL or initials); circular, bleeds to chip edge |
| `removable` | boolean | `false` | shows trailing `✕` |
| `disabled` | boolean | `false` | |
| `label` | string | — | alias of default slot |

- **Slots:** default (label) · `icon` · `avatar`.
- **Interaction:** the chip is clickable (fires `tulpar-click`); when `removable`, the `✕` is an
  **independently focusable `<button aria-label="Remove {label}">`** (separate tab stop) firing
  `tulpar-remove`. Keyboard: Enter/Space activates the chip; Delete/Backspace on the chip removes (when
  removable). **After removal, focus moves** to the next sibling chip or the chip's container (no orphaned focus).
- **Hit area:** the `✕` glyph is 11–14px but its button fills ≈ chip height (negative-margin wrapper) for
  a generous touch target; hover shows a `radius.sm` tonal plate (not a circle).
- **Events:** `tulpar-click` `{}` · `tulpar-remove` `{}`. The `✕` button calls `stopPropagation`, so a
  remove never also emits `tulpar-click`.
- **Reserved (Wave 2):** `selected` / `selectable` and arrow-key group nav land with **ChipGroup** —
  not shipped standalone (a lone selectable chip is an a11y island).

### 4.4 `<tulpar-avatar>` — identity

| Prop | Type | Default | Notes |
|---|---|---|---|
| `src` | string | — | image URL |
| `name` | string | — | used to derive initials + deterministic color + accessible name |
| `initials` | string | — | manual override of derived initials |
| `alt` | string | — | image alt; falls back to `name` |
| `shape` | `rounded-square\|circle` | **`rounded-square`** | |
| `size` | `xs…xl` | **`md`** | |
| `color` | brand family / CSS color | derived | override the deterministic fallback bg |

- **Fallback cascade:** `src` (renders `<img loading="lazy">` with width/height set → no CLS) → on
  `error` or no `src`: **initials** (`initials` prop, else derived from `name`: 1 letter at xs/sm, 2 at
  md+; locale-aware uppercase) → no `name`: **user icon** (`neutral` soft-tonal).
- **Initials typography:** Source Serif 4, weight 600, `letter-spacing:-0.01em` (the distinctive
  "monogram" tell). **Deterministic color:** hash `name` → a **wide** palette of chromatic brand
  families — all families **except** the warning (`ulgen`/amber) and danger (`al`/red) families and the
  neutral/gray families (`colpan·kara·yagiz·boz·mergen·ay`) — so no one is tinted an "error" color while
  variety stays high. `[600]` light / `[500]` dark, on-color text. (Final included list pinned in
  implementation against `primitive/color.ts`.)
- **Slots:** default (custom content, e.g. an icon component) overrides the cascade.
- **Stacking:** standalone now; the **status-dot slot + overlap API are reserved** for AvatarGroup (Wave 2+).
- **A11y:** image → `alt` (or `name`). Initials/icon fallback → `role="img"` + `aria-label="{name}"`;
  decorative avatars beside visible names should be set `aria-hidden` by the consumer.

### 4.5 `<tulpar-skeleton>` — loading placeholder

| Prop | Type | Default | Notes |
|---|---|---|---|
| `variant` | `text\|rect\|circle` | **`text`** | |
| `lines` | number | `1` | text variant; last line shortened (~60%) |
| `width` | string | — | CSS length |
| `height` | string | — | CSS length |
| `radius` | string | per variant | text `radius.sm`, rect `radius.lg`, circle `full` |
| `animation` | `shimmer\|pulse\|none` | **`shimmer`** | |

- Reserves layout (set width/height to avoid CLS). Text line height = `0.7em` (reads as text, not a bar).
- **A11y:** skeletons are `aria-hidden="true"`; the **consumer's container** sets `aria-busy="true"` and
  the spec documents that pattern. Reduced-motion drops to static tint; forced-colors → dashed outline.

### 4.6 `<tulpar-spinner>` — indeterminate loader

| Prop | Type | Default | Notes |
|---|---|---|---|
| `size` | `xs…xl` | **`md`** | |
| `tone` | tone vocab | — | omitted → inherits `currentColor`; set a tone to colorize; brand-green via `tone="custom"` + `color="tulpar"` (custom override, per §3.1) |
| `track` | boolean | `true` | faint full ring behind the arc |
| `delay` | number (ms) | `0` | render nothing until elapsed → no flash for <`delay` ops |
| `label` | string | `"Loading"` | accessible name (alias of slot) |

- **Slots:** `label` (visually-hidden by default).
- **A11y:** `role="status"` + accessible name; reduced-motion → 3-dot fallback; forced-colors → `CanvasText`.

### 4.7 `<tulpar-progress>` — linear + circular

| Prop | Type | Default | Notes |
|---|---|---|---|
| `variant` | `linear\|circular` | **`linear`** | |
| `value` | number | — | omit / `indeterminate` → indeterminate |
| `min` | number | `0` | |
| `max` | number | `100` | |
| `indeterminate` | boolean | `false` | |
| `buffer` | number | — | secondary value (linear only; e.g. "loading more") |
| `tone` | tone vocab | — (brand) | fill defaults to `--tulpar-color-brand-default` (brand green); set a tone to override |
| `state-tone` | boolean | `false` | auto: complete→`success`, `error` state→`danger` |
| `thickness` | `thin\|regular\|thick` | **`regular`** | linear |
| `size` | `sm\|md\|lg` | **`md`** | circular |
| `value-label` | boolean / formatter | `false` | show `42%` or custom `(v,min,max)=>string` |

- **Slots:** `label`.
- **A11y:** `role="progressbar"` + `aria-valuemin/max/now` (+ `aria-valuetext` when `value-label` is a
  formatter, e.g. "Step 3 of 5"); **drop `aria-valuenow` when indeterminate**; `aria-label`/`labelledby`.
- Track `colpan[200]`/`mergen[800]`; fill = tone accent. `state-tone` error is set via attribute/prop by
  the consumer (no implicit failure detection). Reduced-motion per §3.5.

## 5. Distinctiveness (the restrained "Tulpar tells")
1. **Serif monogram avatars** on palette-locked brand-family colors.
2. **Square tags, pill badges** — intentional shape contrast (two object classes).
3. **100° diagonal shimmer** (not opacity pulse).
4. **Knockout surface-color ring** on stacked avatars / corner badges.
5. **Inset-ring chip selection** (Wave 2) + **round-cap quarter-arc spinner** — one "drawn line" motif
   across the loader family.

Guardrails: no gradients on chips, no shadows on inline atoms, no uppercase letter-spaced labels, no
full-circle icon hover plates, no random-HSL avatar colors.

## 7. Tokens (`@tulpar-ui/tokens`)

New `src/semantic/atom.ts` block, bound in `brand/tulpar/{light,dark}.ts`:

```
atom.tone.{neutral,info,success,warning,danger} = {
  soft:    { surface, text, border },     // [100]/[800]/[200] light · [900]/[100]/[700] dark
  outline: { text, border },              // [700]/[400] light · [300]/[500] dark
  solid:   { bg, text },                  // default-step/on-color light · [500]/yagiz950 dark
}
atom.size.{xs..xl}      // height/pad/font/icon/gap/radius (Tag/Chip)
avatar.size.{xs..xl}    // size/initials/radius
```

- `custom` tone reuses the promoted resolver's `-l/-d` inline-var hooks (`--tulpar-{atom}-surface-l` …).
- **`src/semantic/atom.contrast.test.ts`** follows the `feedback.contrast.test.ts` /
  `overlay.contrast.test.ts` pattern but **adds a `solid` assertion** those don't have (toast ships no
  solid): soft text-on-surface ≥4.5, outline text + all borders ≥3:1, **solid text-on-fill ≥4.5** — in
  **both** light and dark. Solid uses a uniform ≥4.5 (no large-text carve-out) to keep the token test
  decoupled from per-atom font sizes.
- Primitive-token lint rule still applies: `*.styles.ts` reference only `--tulpar-atom-*` / semantic vars.

## 8. Accessibility (consolidated, WCAG 2.2 AA)
- **Never color-only:** every tonal Tag/Badge/Chip pairs hue with a dot/icon or text.
- **Loading has a name:** Spinner `role="status"`; Progress `role="progressbar"` + value attrs; skeleton
  containers `aria-busy`. No silent loading for SR users.
- **Remove flow:** `✕` is a labelled button; focus relocates after removal; Delete/Backspace supported.
- **Avatar semantics:** image `alt`/`name`; initials/icon fallback `role="img"`+`aria-label`.
- **Reduced-motion + forced-colors:** explicit per-atom blocks (§3.5–3.6); no shimmer/spin under reduced-motion.
- **Touch targets:** interactive `✕` button ≥ chip height via hit-slop.
- **Focus-visible** rings on all interactive parts, HCM `CanvasText` fallback.
