**Status:** Design (approved direction, pending spec review)

## 1. Summary

Elevate the `@tulpar-ui/shell` sidebar (`tulpar-sidenav` + `tulpar-nav-item`) to an enterprise, product-ready standard (Linear / Vercel / Stripe caliber), wire the official brand assets through the playgrounds, fix a known rail overflow bug, and raise the quality bar of the Storybook stories and the playground demo pages.

This is the v0.7 increment. It is one cohesive effort with two layers:

- **A. Component work** (`@tulpar-ui/shell`, `@tulpar-ui/tokens`, wrappers) — the real design-system deliverable.
- **B. Consumer polish** (playgrounds + docs) — dogfooding + the punch-list the owner raised.

### Approved decisions (from brainstorming)

| Decision | Choice |
|---|---|
| Active-state look | **A — "lit pill" + leading edge-glow** |
| Scope | **Full v0.7 redesign** |

## 2. Color & Token Rules (binding constraint)

This is the hardest constraint and overrides any value suggested during brainstorming:

1. **Primitive color values are frozen.** Do not edit `packages/tokens/src/primitive/color.ts`. `tulpar[500]`, `colpan[50]`, etc. keep their exact current hex.
2. **Shell semantic tokens may be *re-bound* to a different existing primitive** when the new design needs it (e.g. `shell.sidenav.bg: c.colpan[50] → c.white`). Re-binding to an existing primitive is allowed; introducing a new raw hex is not.
3. **New shell tokens** (added to `ShellTokens` / `ShellNavItemTokens`) must resolve to an existing primitive or an `rgba()` of one (the codebase already does this, e.g. `focusRing: "rgba(81, 78, 207, 0.40)"`).
4. **Component CSS** keeps the existing convention: semantic tokens only (`var(--tulpar-shell-…)`), never primitive tokens, with a `var(..., fallback)` whose fallback **mirrors the resolved primitive value** (no arbitrary colors).
5. The ESLint `no-restricted-syntax` primitive-token guard on `*.styles.ts` stays satisfied.

### Color mapping for the new look (all from existing primitives)

| Role | Light | Dark |
|---|---|---|
| Sidenav bg | `c.white` | `c.mergen[950]` (unchanged) |
| Content bg | `c.colpan[100]` (unchanged) or `c.colpan[50]` | `c.mergen[900]` (lift content above nav) |
| Sidenav edge (hairline) | `rgba` of `c.yagiz[950]` ~6% | `rgba` white ~4% |
| Item idle fg | `c.yagiz[700]` (unchanged) | `c.colpan[200]` (unchanged) |
| Item hover bg | `c.colpan[100]` (unchanged) | `c.mergen[900]` (unchanged) |
| **Active bg** (lit pill) | `c.tulpar[50]` (unchanged) | `c.mergen[800]` or `rgba(tulpar[500], .14)` |
| **Active fg** | `c.tulpar[700]` (unchanged) | `c.tulpar[300]` (unchanged) |
| **Active icon / indicator / glow** | `c.tulpar[500]` (unchanged) | `c.tulpar[400]` (unchanged) |
| Section label | `c.kara[500]` | `c.kara[400]` |
| Focus ring | `kam` rgba (unchanged — system-wide consistency) | unchanged |

The lit pill is fully expressible with `tulpar[50]/[500]/[700]` which already exist — no new colors required.

## 3. Component Design

### 3.1 `tulpar-nav-section` (NEW element)

A non-interactive, always-open grouping primitive — distinct from a collapsible group. Fixes the IA hack where the playground nested items under a label-only parent (which renders as an accordion).

- Renders an optional uppercase label + a `role="group"` container with `aria-labelledby` pointing at the label.
- Slots its children (`tulpar-nav-item`s) as siblings, not nested links.
- In **rail** mode the label collapses to a thin divider.
- Data-driven path: `tulpar-sidenav`'s `items` JSON gains a `section: true` (or `type: "section"`) node so the existing `items` API can express sections without children-nesting.
- Registered as `tulpar-nav-section`; exported from `@tulpar-ui/shell`.

### 3.2 `tulpar-nav-item` (enhanced)

Anatomy: `[ icon ] [ label … ] [ trailing ]` with the active edge-glow on the leading inner edge.

- **Active state → lit pill**: drop the separate 3px left bar; active = `bgActive` pill (radius `0.5rem`), `fgActive` text at weight 600, `indicator`-colored icon, and an inset leading edge-glow:
  `box-shadow: inset 2px 0 0 -1px var(--…-indicator), inset 6px 0 8px -6px <indicator @ ~50%>`.
- **Trailing**: add `<slot name="trailing">`; keep convenience props layered on top of it (per the project's "props don't replace slots" rule):
  - `count` (numeric, `font-variant-numeric: tabular-nums`)
  - `dot` / `status` (small colored dot; **requires** an accompanying aria-label — never color-alone)
  - `kbd` (shortcut hint, monospace, **hidden in rail**)
  - existing `badge` retained
  - auto external-link `↗` glyph + `rel="noopener"` when `target="_blank"`.
- Reserve the icon column even when no icon is set, so labels align (no ragged left edge).
- Density driven by a cascaded token (see 3.3 `density`).

### 3.3 `tulpar-sidenav` (enhanced)

- **Layout regions** top→bottom: `header` slot → optional `search` slot → scrollable `nav` → **utility row** → **footer (user/account)**.
- **Header**: hosts the brand lockup (icon + wordmark) + an optional `header-actions` slot + the built-in collapse toggle. Sticky. Collapse toggle emits `tulpar-sidenav-toggle`; rail centers the icon and hides the wordmark.
- **Utility row (NEW)**: a two-up split directly above the footer — left cell = Dark/Light toggle slot, right cell = Configurator button slot. Exposed as `<slot name="utility-start">` / `<slot name="utility-end">` (or a single `<slot name="utility">` the app splits). **In rail mode only the Dark/Light (utility-start) cell is shown**, centered; the Configurator cell is hidden.
- **Footer (user/account)**: `<slot name="footer">` for the avatar + name + chevron block. Sticky bottom. Rail → avatar only, centered.
- **`position` attribute (NEW)**: `"left" | "right"`, default `left`. Drives which side the sidenav sits and mirrors the hairline edge, flyout direction, and active edge-glow side. (`top`/`bottom` explicitly out of scope — see §6.)
- **`density` attribute (NEW)**: `"comfortable" | "compact"`, default `comfortable` (44px floor for a11y; compact ≈ 36–40px).
- **`single-expand` attribute (NEW)**: opt-in exclusive accordion for collapsible groups; default off (multiple groups may stay open).
- Auto-expand the group containing the active route on load; persist the open-group set via the shell's existing `persistKey` mechanism.
- `nav` gets `overflow-x: clip; overflow-y: auto; overscroll-behavior: contain` + top/bottom scroll-shadows.

### 3.4 Rail (collapsed) rewrite

Fixes the **horizontal overflow scrollbar bug** and makes grouped nav reachable when collapsed.

- Root cause: the flyout label is `position:absolute` *inside* the `overflow-y:auto` nav, so its width forces horizontal content → bottom scrollbar.
- Fix: (a) `overflow-x: clip` on `nav`; (b) flyouts escape the scroll container — render via the `popover` API / CSS anchor positioning in the shell's top layer, anchored to the item. This also removes the Chromium-only `:host-context` dependency, fixing Firefox.
- Collapsible **groups** render their children as a **flyout submenu panel** in rail (today they just disappear).
- Flyouts are keyboard-reachable: focus shows the flyout; `ArrowRight` enters a submenu, `Esc`/`ArrowLeft` exits. Open delay ~120ms on hover, instant on focus.
- Flyout direction respects `position` (`right` mirrors to the left).

### 3.5 Accessibility & keyboard

- Roving tabindex (single tab stop for the nav), `ArrowUp`/`Down` (exists) + `Home`/`End` + `ArrowRight`/`Left` to expand/collapse groups.
- `aria-current="page"` on the active leaf only; groups use `aria-expanded` + `aria-controls`.
- Sections: `role="group"` + `aria-labelledby`.
- Color-not-only for `dot`/`status`; reduced-motion guard extended to flyout, edge-glow, scroll-shadow.
- Overlay (mobile) keeps focus-trap + return-focus (shell already handles open/close + Esc).

### 3.6 The signature detail (motion)

On a nav item becoming active (route change), the leading edge-glow **wipes once** from the leading edge before settling (~260ms), echoing the brand mark's ascending strokes. Pure CSS pseudo-element; falls back to a plain fade under `prefers-reduced-motion`. Subtle, fast, brand-meaningful.

### 3.7 Wrappers (Angular + Vue)

- Angular wrapper: expose new inputs (`position`, `density`, `singleExpand`), the new slots, and `tulpar-nav-section` as `tulpar-nav-section-ng`. **Signals only** (no RxJS), per project rule.
- Vue wrapper: matching props/slots + `TulparNavSection`.
- Smoke tests only (class exported, defaults correct), per the wrapper testing convention.

## 4. Brand Assets (consumer)

- Add the official kit assets to each consumer's `public/brand/`: `TulparUi-icon.svg`, `TulparUi-wordmark.svg`, plus the lockups already present. Apply to `apps/playground-ng`, `apps/playground-vue`, `apps/docs`.
- **Dark wordmark**: the supplied `TulparUi-wordmark.svg` is dark-ink (`#07291F`) and disappears on dark backgrounds. Provide a light/white-text wordmark variant for dark mode (derive from the kit; the dark text color comes from the same brand ink, recolored to a light neutral — not a new brand color).
- **Sidebar header** shows the brand lockup (icon + wordmark), swapping the wordmark asset by theme. Rail → icon only.
- **Topbar (when used)**: the menu button uses `TulparUi-icon.svg` (replacing the generic hamburger).

## 5. Playground & Docs Polish (punch-list)

### 5.1 Playground menu IA
- Replace the single nested "Components" parent with real **sections**: e.g. `Components` (Button, TextInput, Textarea, NumberInput) and `Foundations` (Colors, …). **Colors leaves the Components group.**
- Add an **icon to every nav item** (the menu currently has none).
- Wire the footer utility row: Dark/Light toggle (moved out of the aside) + Configurator placeholder button; user/account block at the very bottom.

### 5.2 Footers
- Redesign the careless single-line footers in both playgrounds into a deliberate, on-brand footer (links / version / copyright), consistent light + dark.

### 5.3 Demo pages
- Give each demo page (button, text-input, textarea, number-input, colors) a deliberate layout: hero/title, anatomy, interactive examples, real-world composition — not a bare component dump. Both Angular and Vue playgrounds.

### 5.4 Storybook stories
- **Fix the Button `Default` render bug**: it currently binds only `severity/variant/size/shape/disabled/loading`, so the `color`, `icon-position`, `justify`, `raised`, `block`, `iconOnly`, `icon-separator` controls do nothing ("renk seçimi çalışmıyor"). Wire **all** declared `argTypes` into the `Default` render.
- Deepen each component's stories: a true interactive playground story + real-world composition examples, beyond the current shallow one-offs.
- Confirm the `Shell` story demonstrates the new sidebar (sections, lit pill, rail flyout, utility row, position left/right).
- Confirm `Colors` lives under `Foundations/` in Storybook (it already does) — the IA fix is the playground menu, not the story title.

## 6. Out of Scope (v0.7)

- `position: top` / `bottom` (horizontal nav) — needs a separate horizontal-layout engine; deferred. Already flagged in the v0.6 spec.
- Real **Configurator panel** (v2 — needs Select/Switch/Checkbox).
- Workspace switcher, user-menu **dropdown** behavior (only the slot/affordance ships).
- Command/search **component** (only the `search` slot ships).
- Typeahead nav, 3+ nesting depth, RTL flyout-mirroring polish, second brand theme.

## 7. Risks & Notes

- **`popover`/anchor positioning** browser support: target is Chromium (WTR), but provide a graceful fallback for the flyout so it degrades without breaking.
- **Token re-binding** must be verified against existing `light.test.ts` / `dark.test.ts` snapshots; update tests deliberately, never loosen the primitive-value freeze.
- **Dark wordmark** is a new asset deliverable — confirm the recolor reads as the same brand, not a new color.
- Keep each version tightly scoped per CLAUDE.md; surface any pull toward out-of-scope items rather than absorbing them.
