# Tulpar UI v0.1 — Button Deep Dive Design Spec

**Date:** 2026-06-05
**Status:** Approved (brainstorm complete, ready for implementation plan)
**Scope:** v0.1 — Button component + foundational architecture

---

## 1. Overview & Goals

Tulpar UI, hem Angular hem Vue projelerinde kullanılabilen, sıfırdan tasarlanmış bir UI component library'sidir. PrimeNG ve PrimeVue'ye bağımlılığı yoktur; kendi token sistemi, davranış kuralları ve component'leri vardır.

**v0.1 hedefi:** Mimari foundation'ı (paket yapısı, token sistemi, theming, build pipeline, demo akışı) tam olarak kurmak ve bunu **tek bir component (Button)** üzerinden derinlemesine ispatlamak. Sonraki component'ler bu şablonu izleyerek eklenir.

**Tasarım ilkeleri:**

- **Web standards first:** Lit tabanlı Web Components — framework-agnostik, browser native
- **Yavaş büyüme:** Önce mimari ve tek component. Doğru pattern'leri yerleştir; sonra ölçekle
- **Multi-brand ready:** v0.1'de tek brand var (`tulpar`), ama mimari multi-brand ve light/dark için baştan kurulur
- **Enterprise defensive:** `tulpar-` prefix (kısaltma yok), 3-katmanlı token mimarisi, semantic versioning, no surprises

**Out of scope (v0.1'de yok):**

- Birden fazla component (Input, Checkbox, Select vs. — v0.2+'a)
- 2. brand teması (mimari hazır, ama sadece `tulpar` brand'i implement)
- React wrapper'ı (gelecekte düşünülebilir, v0.1'de yok)
- SSR optimizasyonu (Lit zaten SSR-uyumlu ama özel tuning yok)
- Mobile-native (React Native / Flutter) hedefi yok

**Button'un gelişmiş varyantları (v0.2+):**

Bunlar v0.1'de implement edilmiyor ama mimari API tasarımında "ileride bu eklenebilir mi" diye düşünülerek karar veriliyor:

- **Split button** — sol kısmı normal action, sağ kısmı dropdown trigger. İki bağımsız tıklanabilir alan. Klavye davranışı kompleks.
- **Dropdown / menu button** — tek tıklamayla menu açan buton (`<tulpar-menu-button>` ileride). Listbox / menu component'i ile birlikte gelir.
- **Toggle button** — pressed/unpressed state'i olan, `aria-pressed` taşıyan buton. İlke olarak `<tulpar-button>`'a `?pressed` attribute ekleyerek desteklenebilir ama v0.1'de yok.
- **Button with badge/counter** — buton üstünde rozet/sayaç gösteren. `<tulpar-badge>` component'i ile birlikte gelir, kompozisyon yoluyla çözülür.
- **Icon button with tooltip** — tooltip component'i ile birlikte gelir. v0.1'deki icon-only mode sadece `aria-label` ile yetiniyor; tooltip yok.
- **FAB (Floating Action Button)** — Material Design pattern'i. Position: fixed davranışı v0.1 dışı; gerekiyorsa `<tulpar-fab>` ayrı component olarak gelir.
- **Loading state inline content variation** — şu an loading sadece spinner gösteriyor + label gizleniyor; ileride "spinner + custom loading text" senaryosu istenirse `slot="loading"` eklenebilir.

## 2. Architecture

### High-Level Diagram

```
┌─────────────────────────────────────────────────┐
│  @tulpar-ui/tokens                              │
│  - TypeScript token definitions                 │
│  - Build step: TS → CSS custom properties       │
│  - Zero runtime, zero dependencies              │
└─────────────────────────────────────────────────┘
                       ▲
                       │ peer dep
                       │
┌─────────────────────────────────────────────────┐
│  @tulpar-ui/core (Lit Web Components)           │
│  - <tulpar-button>, <tulpar-button-group>       │
│  - Component CSS uses semantic tokens only      │
│  - SOURCE OF TRUTH for behavior + rendering     │
└─────────────────────────────────────────────────┘
                       ▲
              ┌────────┴────────┐
              │                 │
┌──────────────────────┐ ┌──────────────────────┐
│ @tulpar-ui/angular   │ │ @tulpar-ui/vue       │
│ Thin wrappers:       │ │ Thin wrappers:       │
│ - ControlValueAcc.   │ │ - v-model            │
│ - Signal-based I/O   │ │ - props/emits        │
│ - Angular types      │ │ - Vue types          │
└──────────────────────┘ └──────────────────────┘
```

### Why this layering

- **`tokens` paketi ayrı:** Hem Web Components hem ileride tema-only kullanıcılar import edebilir. Sıfır dependency = sıfır risk.
- **`core` paketi source of truth:** Tüm davranış, render, a11y burada. Angular ve Vue wrapper'ları bu Web Component'i sarmalıyor — davranışı yeniden yazmıyor.
- **Wrapper'lar gerçekten ince:** Sadece framework idiomlarına bağlama yapıyor. Bug fix tek yerde (core), wrapper'lara otomatik akıyor.

### Karar gerekçeleri (kayıt için)

| Karar | Seçim | Alternatifler | Neden |
|---|---|---|---|
| Cross-framework yaklaşım | Web Components (Lit) | Stencil; shared TS core + wrappers; sadece tokens | Enterprise standard (IBM Carbon, Adobe Spectrum, Google Material). Tek source of truth. Web standards. |
| WC kütüphanesi | Lit | Stencil, vanilla, FAST | Daha küçük (~5KB), daha hızlı, standartlara yakın, Lit ekosistemi sağlam |
| Element prefix | `tulpar-` | `t-`, `tu-`, `tui-`, `tlp-` | Collision-safe (Toast UI çakışmasından kaçınma), branding, grep-friendly |
| Token mimarisi | 3-layer (primitive/semantic/brand+mode) | Sadece CSS vars; sadece TS const | TS type-safety + IDE autocomplete + multi-brand + light/dark açık eksenler |
| Dark mode trigger | `.dark` class | `prefers-color-scheme`, `[data-theme]` | Berdan ve diğer Tailwind-uyumlu projelerle uyumluluk |
| Brand selector | `[data-brand="..."]` | CSS class | Multi-brand'i mode'dan ortogonal tutmak, CSS specificity'i temiz tutmak |

## 4. Token System (3-Layer Multi-Brand)

### Layer 1: Primitive Tokens

Ham değerler. Hiçbir semantik anlam yok. Renk paleti, spacing scale, typography scale, shadow scale, border-radius scale.

```ts

    50:  "#eff6ff",
    100: "#dbeafe",
    // ...
    500: "#3b82f6",
    600: "#2563eb",
    900: "#1e3a8a",
  },
  gray: {
    50:  "#f9fafb",
    // ...
    900: "#111827",
  },
  white: "#ffffff",
  black: "#000000",
  // red, green, orange, yellow, vs.

// packages/tokens/src/primitive/spacing.ts

  0: "0",
  1: "4px",
  2: "8px",
  3: "12px",
  4: "16px",
```


### CSS Generation Pipeline

`packages/tokens/build/generate-css.ts` build script'i:

1. Tüm primitive'leri `:root` altında CSS custom property'leri olarak yazıyor (ön ek: `--tulpar-primitive-`)
2. Her brand × mode kombinasyonu için bir selector bloğu üretiyor:

```css
/* Primitives — always available */
:root {
  --tulpar-primitive-blue-600: #2563eb;
  --tulpar-primitive-gray-900: #111827;
  --tulpar-primitive-white: #ffffff;
  /* ... */
}

/* Tulpar brand, light mode (default) */
:root,
[data-brand="tulpar"] {
  --tulpar-color-bg-surface: var(--tulpar-primitive-white);
  --tulpar-color-text-primary: var(--tulpar-primitive-gray-900);
  --tulpar-color-brand-default: var(--tulpar-primitive-blue-600);
  --tulpar-color-brand-hover: var(--tulpar-primitive-blue-700);
  --tulpar-color-brand-on-brand: var(--tulpar-primitive-white);
  /* ... */
}

/* Tulpar brand, dark mode */
.dark,
[data-brand="tulpar"].dark,
.dark [data-brand="tulpar"] {
  --tulpar-color-bg-surface: var(--tulpar-primitive-gray-900);
  --tulpar-color-text-primary: var(--tulpar-primitive-gray-50);
  --tulpar-color-brand-default: var(--tulpar-primitive-blue-400);
  /* ... */
}
```

### Token Naming Convention

- CSS prefix: `--tulpar-` (collision-safe)
- Primitives: `--tulpar-primitive-<category>-<scale>` (e.g., `--tulpar-primitive-blue-600`)
- Semantic: `--tulpar-<category>-<subcategory>-<role>` (e.g., `--tulpar-color-bg-surface`)
- TS exports: camelCase (`tulparLight.color.bg.surface`), generated CSS: kebab-case

## 5. Button Component Deep Dive

### Element Names

- `<tulpar-button>` — single button
- `<tulpar-button-group>` — sibling button container

### API

```html
<tulpar-button
  variant="primary"        <!-- primary | secondary | danger | ghost | link -->
  size="md"                <!-- xs | sm | md | lg | xl | 2xl | 3xl -->
  type="button"            <!-- button | submit | reset -->
  ?disabled                <!-- boolean attribute -->
  ?loading                 <!-- boolean attribute, shows spinner, aria-busy -->
  ?icon-only               <!-- boolean attribute, requires aria-label -->
  as="button"              <!-- button | a -->
  href=""                  <!-- if as="a" -->
  target=""                <!-- if as="a" -->
  rel=""                   <!-- if as="a" -->
  aria-label=""            <!-- required when icon-only -->
>
  <tulpar-icon slot="start">...</tulpar-icon>  <!-- leading icon -->
  Button Label
  <tulpar-icon slot="end">...</tulpar-icon>    <!-- trailing icon -->
</tulpar-button>
```

> **Not:** `<tulpar-icon>` v0.1'de implement edilmiyor. Slot'a herhangi bir HTML/SVG verilebiliyor. Icon component v0.2+ konusu.

### Variants

| Variant | Use case | Visual |
|---|---|---|
| `primary` | Primary action ("Save", "Submit") | Brand background, white text |
| `secondary` | Secondary action ("Cancel") | Border, transparent bg, brand text |
| `danger` | Destructive action ("Delete") | Red background, white text |
| `ghost` | Tertiary action, low emphasis | Transparent bg, no border, hover bg |
| `link` | Inline link styling | Underline, no padding, brand text |

### Sizes

| Size | Height | Padding-X | Font-size | Icon size | Kullanım örneği |
|---|---|---|---|---|---|
| `xs` | 24px | 8px | 12px | 12px | Tablo içi inline action'lar, badge'e yakın küçük buton |
| `sm` | 32px | 12px | 14px | 14px | Compact form'lar, sidebar action'ları |
| `md` | 40px | 16px | 14px | 16px | Default — çoğu form ve action |
| `lg` | 48px | 20px | 16px | 18px | Önemli CTA, ödeme/onay |
| `xl` | 56px | 24px | 18px | 20px | Hero CTA, marketing pages |
| `2xl` | 64px | 28px | 20px | 24px | Mobile-first büyük dokunma alanı, splash |
| `3xl` | 72px | 32px | 24px | 28px | Landing page primary action, daha düşük frekansta kullanım |

`icon-only` modunda padding tüm yönlerde eşitlenir (square button), height = width. Default size hâlâ `md`.

> **Not:** Size token'ları `@tulpar-ui/tokens` paketinde `semantic.button.size.{xs|sm|md|lg|xl|2xl|3xl}` altında tanımlanır. CSS custom property üretildiğinde: `--tulpar-button-size-xs-height`, `--tulpar-button-size-2xl-height` vs. — sayısal prefix CSS var isimlerinde sorun değil (`--tulpar-button-size-2xl-*` valid identifier).

### States

| State | Detection | Behavior |
|---|---|---|
| Default | — | Variant base styles |
| Hover | `:hover` | Darker brand color, no layout shift |
| Focus-visible | `:focus-visible` | Focus ring (`box-shadow` ile, layout shift olmaz) |
| Active | `:active` | Even darker variant color |
| Disabled | `disabled` attribute | Reduced opacity, `cursor: not-allowed`, no pointer events. Form-associated → form submit'i tetiklemiyor |
| Loading | `loading` attribute | Spinner gösteriliyor, label gizleniyor (width korunuyor → layout shift olmuyor), `aria-busy="true"`, `pointer-events: none` |

### Form Integration (ElementInternals)

Form-associated custom element olarak implement edilir:

```ts
class TulparButton extends LitElement {
  static formAssociated = true;
  private _internals = this.attachInternals();

  private handleClick(e: MouseEvent) {
    if (this.disabled || this.loading) {
      e.preventDefault();
      e.stopImmediatePropagation();
      return;
    }
    if (this.type === "submit") {
      this._internals.form?.requestSubmit();
    } else if (this.type === "reset") {
      this._internals.form?.reset();
    }
  }
}
```

Bu, `<form>` içine konduğunda `<button type="submit">` davranışını taklit eder — Angular Reactive Forms ve Vue native form binding bunu doğal olarak alır.

### Polymorphism (`as="a"`)

Eğer `as="a"` veya `href` verilmişse, internal render `<a>` etiketi olur. Disabled durumda `<a>` için `aria-disabled="true"` + `tabindex="-1"` + `pointer-events: none` kombinasyonu, çünkü HTML `<a>` element'inin native disabled state'i yok.

### Accessibility Checklist

- [ ] Role: native `<button>` veya `<a>` kullanıldığı için role implicit
- [ ] Keyboard: Enter ve Space (button), Enter (a) ile aktif
- [ ] Focus-visible: sadece klavye odağında ring (mouse'da değil)
- [ ] Focus ring: `box-shadow` ile, asla `outline: none` yapmıyoruz
- [ ] Loading: `aria-busy="true"`, spinner için `aria-label="Loading"`
- [ ] Disabled: `disabled` attribute (form'a etki), screen reader otomatik anlıyor
- [ ] Icon-only: `aria-label` zorunlu (dev mode console warn, prod no-op)
- [ ] Color contrast: tüm variant'lar WCAG AA (4.5:1 text, 3:1 borders)
- [ ] `prefers-reduced-motion`: spinner ve transition'lar bu setting'e saygı duyuyor

### Button Group

```html
<tulpar-button-group>
  <tulpar-button variant="secondary">Day</tulpar-button>
  <tulpar-button variant="secondary">Week</tulpar-button>
  <tulpar-button variant="secondary">Month</tulpar-button>
</tulpar-button-group>
```

Davranış:
- Sibling button'ların border-radius'unu akıllıca yönetir (ilk: sol radius, son: sağ radius, ortalar: 0)
- Border'ları çiftleşmez (border-right: 0 hilesi)
- Klavye navigation: yatay arrow keys (ArrowLeft / ArrowRight) ile sibling button'lar arasında focus geçişi, Home/End ile ilk/son button, roving tabindex pattern (sadece odaktaki button `tabindex="0"`, diğerleri `tabindex="-1"`)
- `role="group"` ile ARIA grup semantiği

```css

}
```

## 6. Framework Wrapper Strategy

### Angular Wrapper (`@tulpar-ui/angular`)

> Aşağıdaki örnekte selector `tulpar-button-ng` olarak gösteriliyor. Bu seçim **tentative** — final selector kararı section 9'da open decision olarak duruyor (`tulpar-button-ng` vs `tulpar-button` + `CUSTOM_ELEMENTS_SCHEMA`).

```ts
// button.component.ts
import {
  Component,
  ChangeDetectionStrategy,
  input,
  output,
  CUSTOM_ELEMENTS_SCHEMA,
} from "@angular/core";
import "@tulpar-ui/core/button";

@Component({
  selector: "tulpar-button-ng",
  template: `
    <tulpar-button
      [attr.variant]="variant()"
      [attr.size]="size()"
      [attr.type]="type()"
      [attr.disabled]="disabled() ? '' : null"
      [attr.loading]="loading() ? '' : null"
      (click)="clicked.emit($event)"
    >
      <ng-content select="[slot=start]" />
      <ng-content />
      <ng-content select="[slot=end]" />
    </tulpar-button>
  `,
  changeDetection: ChangeDetectionStrategy.OnPush,
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
})
export class TulparButtonComponent {
  variant = input<"primary" | "secondary" | "danger" | "ghost" | "link">("primary");
  size = input<"xs" | "sm" | "md" | "lg" | "xl" | "2xl" | "3xl">("md");
  type = input<"button" | "submit" | "reset">("button");
  disabled = input(false);
  loading = input(false);
  clicked = output<MouseEvent>();
}
```

Notlar:
- Signal-based input/output (Angular 17+ idiom, kullanıcının `feedback_use_signals.md` tercihiyle uyumlu)
- `CUSTOM_ELEMENTS_SCHEMA` kullanılıyor çünkü iç template'te `<tulpar-button>` Web Component'i var
- Form integration: Web Component zaten form-associated; Angular FormControl ile özel bir köprü gerekmiyor (gerekirse v0.2'de `ControlValueAccessor` opsiyonel olarak eklenir, ancak Button için zaten input form alanı değil — submit button olduğu için ekstra binding gereksiz)
- Selector farkı: Angular tarafında `<tulpar-button-ng>` kullanılıyor (Web Component ile çakışmayı önlemek için). Alternatif: aynı selector'ı kullan ve schemas ile Web Component'i tanı. Implementation plan'da karar.

### Vue Wrapper (`@tulpar-ui/vue`)

```vue
<!-- TulparButton.vue -->
<script setup lang="ts">
import "@tulpar-ui/core/button";

interface Props {
  variant?: "primary" | "secondary" | "danger" | "ghost" | "link";
  size?: "xs" | "sm" | "md" | "lg" | "xl" | "2xl" | "3xl";
  type?: "button" | "submit" | "reset";
  disabled?: boolean;
  loading?: boolean;
}

const props = withDefaults(defineProps<Props>(), {
  variant: "primary",
  size: "md",
  type: "button",
  disabled: false,
  loading: false,
});

const emit = defineEmits<{
  click: [event: MouseEvent];
}>();
</script>

<template>
  <tulpar-button
    :variant="variant"
    :size="size"
    :type="type"
    :disabled="disabled || null"
    :loading="loading || null"
    @click="emit('click', $event)"
  >
    <slot name="start" />
    <slot />
    <slot name="end" />
  </tulpar-button>
</template>
```

Notlar:
- Vue'nun `compilerOptions.isCustomElement: tag => tag.startsWith('tulpar-')` ayarı consumer tarafında olacak (docs'ta belirtilir)
- Boolean attribute'ları `false → null` yaparak attribute'ı tamamen kaldırıyoruz (Web Components boolean attribute davranışı için zorunlu)

### Wrapper Tasarım Prensipleri

1. **Wrapper YENİ DAVRANIS EKLEMEZ.** Sadece framework'ün idiomatik API'sini Web Component'e çeviriyor.
2. **Wrapper, Web Component'i import ediyor.** Yan etki olarak `customElements.define()` çağrılıyor.
3. **Wrapper API yüzeyi minimal.** Variant/size gibi property'ler doğrudan attribute olarak geçiyor.
4. **Slot mapping doğal.** Angular `ng-content`, Vue `<slot>` ile Web Components slot'larına bağlanıyor.

## 9. Open Decisions (Implementation Plan'da Karar Verilecek)

Bunlar mimari kararı değiştirmiyor ama implementation sırasında bir tercih gerekecek:

1. **Component test runner** — `@web/test-runner` (Lit standard, real browser) vs Vitest + happy-dom (hızlı, daha kolay setup). İlki Lit best practice, ikincisi monorepo'da tutarlı tooling.
2. **Angular wrapper selector** — `<tulpar-button-ng>` (çakışma yok ama tutarsız) vs `<tulpar-button>` (tutarlı ama schema gerekliliği)
3. **Vite kütüphane modunda CSS bundling** — CSS'i ayrı dosyada mı (tüketici manuel import) yoksa JS'e mi gömülü (otomatik) — Lit'in `css` tagged template kullanımı zaten shadow DOM içinde, ama tokens CSS'i global olduğu için ayrı dosya muhtemelen doğru
4. **Lit version** — Lit 3 (stable, 2026'da hâlâ aktif)
5. **TS path mapping vs gerçek build artifact'ler** — geliştirme sırasında source'a doğrudan TS path mapping, publish için derlenmiş artifact'ler
6. **Storybook'taki tokens nasıl yükleniyor** — `@tulpar-ui/tokens`'tan üretilen CSS'i Storybook preview head'ine inject etmek
