# Tulpar UI v0.2 — Design System Redesign

**Status:** Approved 2026-06-08
**Predecessor:** [v0.1 — Button design](./2026-06-05-tulpar-ui-v0.1-button-design.md)
**Owner:** Kaan Akcan

---

## 1. Overview

v0.2 replaces the entire design token foundation of Tulpar UI with a new "Financial-direction" identity (Navy + Gold + warm cream) and a Tailwind-equivalent chromatic spectrum (17 chromatic families + 2 neutrals + 2 brand). Typography moves from Inter to Source Serif 4 (display) + Source Sans 3 (UI/body). Shadows are navy-tinted. Border radius is conservative.

**Scope of change:**
- `@tulpar-ui/tokens` package: completely rebuilt (new primitives, new brand bindings).
- `@tulpar-ui/core` component CSS: token references updated where names change; structural code untouched.
- `@tulpar-ui/angular` and `@tulpar-ui/vue` wrappers: no code change (pass-through props).
- `apps/docs`, `apps/playground-ng`, `apps/playground-vue`: no code change beyond auto-regenerated CSS.

**Visual character to convey:** authority, longevity, premium restraint — the look of a private banking dashboard meeting a modern dev tool.

## 2. Goals & Non-Goals

**Goals:**
- Establish a distinctive brand identity (Navy + Gold) that separates Tulpar from generic Tailwind-aligned design systems.
- Expand the chromatic palette to Tailwind-equivalent breadth so consumers can build any UI without reaching outside Tulpar.
- Refine the visual character (Source Serif display, navy-tinted shadows, conservative radius) for an enterprise feel.
- Preserve the v0.1 semantic API surface — components consume the same semantic tokens, only values change.

**Non-Goals (binding):**
- No new components. Button + ButtonGroup remain the only components. Input, Checkbox, Select, Modal, Tooltip, FAB, Split Button, etc. → out of scope.
- No new Button variants. The 5 existing variants (`primary`, `secondary`, `danger`, `ghost`, `link`) stay. Gold/Success/Warning button variants are NOT shipped; their tokens exist at the primitive layer for future v0.3 work.
- No second brand. Tulpar is the only brand.
- No SSR/RSC optimization beyond what Lit provides natively.
- No font bundling. Consumers are responsible for loading Source Serif 4 / Source Sans 3 / JetBrains Mono (recommended CDN snippet provided in README).

## 3. Brand Identity

Tulpar — the winged horse of Turkic mythology. Mythic, powerful, far-reaching. The v0.2 identity translates that into:

| Aspect | Choice | Rationale |
|---|---|---|
| Primary brand color | Navy `#0a2540` | Authority, financial gravity, ancient resonance. Same hue Stripe uses for its marketing dark. |
| Premium accent | Gold `#b8985a` | Muted gold (not yellow-gold) — heritage, prestige. Used sparingly for marketing CTAs and brand moments, not general UI. |
| Default surface | Warm cream `#fafaf9` (stone-50) | Paper feel, warmer than pure white, easier on long reading sessions. |
| Display type | Source Serif 4 700 | Editorial elegance for headlines. |
| UI/body type | Source Sans 3 400/500 | Humanist sans — readable, friendly, premium. |
| Monospace | JetBrains Mono | Code, numbers, tabular data. |

## 4. Color System

### 4.1 Architecture (unchanged from v0.1)

Three layers:
1. **Primitive** — raw values (`navy.700`, `red.600`, `stone.100`).
2. **Semantic** — anlamsal roles (`color.brand.default`, `color.text.primary`, `color.danger.hover`).
3. **Brand × Mode** — semantic → primitive bindings for `tulpar/light` and `tulpar/dark`.

CSS output: same as v0.1 — `:root` for primitives + Tulpar light, `.dark` for dark mode, `[data-brand="tulpar"]` for explicit brand scoping.

## 5. Typography

### 5.1 Font Families

| Role | Family | Stack |
|---|---|---|
| `font.family.display` | Source Serif 4 | `"Source Serif 4", "Source Serif Pro", Georgia, serif` |
| `font.family.ui` (was `body` in v0.1) | Source Sans 3 | `"Source Sans 3", "Source Sans Pro", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |
| `font.family.mono` | JetBrains Mono | `"JetBrains Mono", "Fira Code", ui-monospace, SFMono-Regular, monospace` |

> **Note:** `font.family.body` is renamed to `font.family.ui` to reflect that Tulpar provides a single UI font (Source Sans 3), with the display serif as a separate role. This is a semantic-layer rename; a backwards-compatible alias is NOT provided because v0.1 has not been published.

**Font loading (consumer responsibility):** Tulpar UI does not bundle fonts. Recommended CDN snippet (will be added to README and Storybook `preview-head.html`):

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Source+Serif+4:wght@500;600;700;800&family=Source+Sans+3:wght@400;500;600;700&family=JetBrains+Mono:wght@400;500&display=swap"
  rel="stylesheet"
/>
```

## 10. Component Impact

### 10.1 Button (`<tulpar-button>`)

**No API changes.** Same 5 variants, same 7 sizes, same props.

**Visual changes (purely token-driven):**
- Default font-family: Source Sans 3 (was Inter via `font.family.body`)
- Border radius: 4px (was 6px) — uses `--tulpar-button-border-radius` which inherits from `radius.md`
- Primary variant background: navy.700 `#0a2540` (was blue.600 `#2563eb`)
- Focus ring: navy@40% (was blue@50%)
- Disabled state opacity: unchanged (0.5)
- Hover/active timing: unchanged (200ms standard easing)

**The CSS file `tulpar-button.styles.ts` will be updated to reference any renamed tokens** (notably `font.family.body` → `font.family.ui`). All other selectors remain.

### 10.2 ButtonGroup (`<tulpar-button-group>`)

**No changes.** Group-position styling (border-radius collapse + 1px margin overlap) reads from button's `radius.md` and works regardless of the new value.

### 10.3 Other Consumers

- Storybook stories: no story changes. **`apps/docs/.storybook/preview-head.html` gets a Google Fonts `<link>` for Source Serif 4 + Source Sans 3 + JetBrains Mono** (matches the snippet in §5.1).

## 11. Migration & Breaking Changes

v0.1 has not been published to npm. v0.2 supersedes v0.1 entirely. The migration is internal-only.

**Token-level breaking changes:**
- Primitive color families: `blue`, `red`, `green`, `yellow`, `orange`, `gray` → expanded set (see §4). Old names that overlap (`blue`, `red`, `green`, `yellow`, `orange`) keep their Tailwind-aligned hex values. `gray` is **renamed to `stone`** (or use `slate` for cool alternative). Numeric `950` stop is added to all chromatic families (v0.1 only went 50–900).
- Brand primitive `gray` no longer exists; semantic mappings that referenced `gray.900` etc. are repointed to `stone.900`.
- New primitives: `navy`, `gold`, `stone`, `slate`, `orange` (already existed), `amber`, `yellow`, `lime`, `emerald`, `teal`, `cyan`, `sky`, `indigo`, `violet`, `purple`, `fuchsia`, `pink`, `rose`.
- Typography primitive: `fontFamily.body` is **renamed to `fontFamily.ui`**. New `fontFamily.display` added.
- Type scale: adds `5xl`, `6xl`, `7xl`. Existing scale unchanged for `xs–4xl`.
- Font weights: adds `heavy` (800).
- Line heights: adds `tight` (1.05) and `snug` (1.2). v0.1's `tight` (1.2) becomes v0.2's `snug`. v0.1 consumers used `lineHeight.tight` for headlines — now `lineHeight.snug` for the same effect.
- Border radius: `md` `6px → 4px`, `lg` `8px → 6px`. Bigger sizes (`xl`/`2xl`/`full`) unchanged.
- Shadow values: all shadows go from black-tinted to navy-tinted (rgba color change; structure identical).

**Semantic-level changes:**
- `color.bg.*`, `color.text.*`, `color.border.*`, `color.brand.*`, `color.danger.*`, `color.success.*`, `color.warning.*`, `color.neutral.*`: **same names**, repointed to new primitives.
- `color.focusRing`: same name, new value (navy@40% instead of blue@50%).
- `font.family.body` → `font.family.ui` (rename, no alias).
- `font.family.display`: new.

**Component-level changes:** None at the API surface. CSS file updates reference renamed tokens.

**Generated CSS file naming:** Unchanged. `tulpar.css`, `tulpar-light.css`, `tulpar-dark.css` still produced by `packages/tokens/build/generate-css.ts`. Consumer import paths unchanged.
