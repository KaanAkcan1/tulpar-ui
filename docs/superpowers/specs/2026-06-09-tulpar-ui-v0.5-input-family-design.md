# Tulpar UI v0.5 — Input Family Design

**Status:** Draft
**Author:** Kaan Akcan
**Date:** 2026-06-09
**Target release:** `@tulpar-ui/*` v0.5.0 (minor pre-1.0 bump — 3 new components, no breaking changes to existing Button surface)

## 1. Background & motivation

After Button (v0.1–v0.4) the next foundational primitives are the **Input family**: text-style fields that drive forms across enterprise apps. The eventual targets in the long-term roadmap (AG Grid-equivalent data grid, Gantt) sit on top of these primitives — inputs must work both as standalone form fields and as cell editors inside dense grid layouts.

Every decision in this spec is grounded in a cross-industry survey of 10 enterprise design systems: PrimeNG, Ant Design, Carbon, Fluent UI v9, Material UI, Adobe Spectrum, Atlassian Design System, Mantine, Vaadin, SLDS. Per-system findings and the synthesis behind specific choices live in the brainstorming transcript; this spec records the conclusions only.

## 2. Scope

### In scope (v0.5)

Three new public custom elements, sharing an unexported `FormFieldBase`:

1. **`<tulpar-text-input>`** — single-line text. Covers `type ∈ text | email | url | tel | search | password` as variants. Includes built-in input mask.
2. **`<tulpar-textarea>`** — multi-line text, autosize default.
3. **`<tulpar-number-input>`** — numeric input with `Intl.NumberFormat`-based locale formatting + steppers.

### Out of scope (deferred)

- **`<tulpar-password-input>`** as a dedicated component (with strength meter). v2 covers password via `<tulpar-text-input type="password">` + auto reveal toggle.
- **`<tulpar-phone-input>`** with international country selector + libphonenumber formatting. Out of core — would ship as a separate package later if demand justifies it. `type="tel"` covers basic.
- **`<tulpar-search-input>`** as a dedicated component. v2 covers search via `<tulpar-text-input type="search">` + auto search icon + auto clearable.
- **`<tulpar-email-input>`**, **`<tulpar-url-input>`** — `type` variants only, never separate components (5 of 10 surveyed systems agree).
- **Optional mask tokens** (`?` modifier, `[optional]` blocks). v2 supports only required-or-literal tokens.
- **Big-int / arbitrary precision mode** (AntD `stringMode`). v2 uses JS native `number`. Crypto/financial-grade precision is a v3+ concern.
- **`<tulpar-form-field>` wrapper** for external anatomy composition. v2 input components own their own label/helper/error.
- **`setValidity` custom validity API** — deferred to a future form-validation surface (consistent with v0.4 out-of-scope note).
- **React wrapper**.

```
@tulpar-ui/core

    number-input/
      tulpar-number-input.ts
      tulpar-number-input.styles.ts
      tulpar-number-input.test.ts
```

### 3.1 FormFieldBase responsibilities

`FormFieldBase` is **unexported**, abstract, extends `LitElement`. Per the existing input architecture decision: internal DRY without public inheritance.

Owns:

- Form-association plumbing (`static formAssociated = true`, `attachInternals()`, `name` / `value` reflection, `setFormValue`).
- Anatomy: `label` (attribute + slot), `label-position`, `helper-text`, `error-text`, `warn-text`, `required`, `necessity-indicator`.
- Status taxonomy: `invalid` / `warn` (validity, exclusive) and `validating` (process, orthogonal).
- Sizing: `size ∈ xs | sm | md | lg | xl`.
- Visual variant: `variant ∈ outlined | filled | underlined | ghost`.
- Disabled / readonly state.
- Prefix / suffix slot host markup + pointer-events plumbing.
- Auto-suffix lane (status icon, copy, paste, clear, password reveal, steppers — managed per subclass).
- Message row layout-shift-safe reservation + `no-message-space` opt-out.
- Width: `:host { display: block; width: 100% }`.
- ID generation + ARIA wiring (`aria-describedby`, `aria-invalid`, `aria-required`, `role="alert"` on error region).

Subclasses opt into which auto-suffix affordances they expose. E.g., NumberInput exposes steppers; Textarea does not expose prefix/suffix slots.

### 3.2 Public components

Each public component:

- Extends `FormFieldBase` internally.
- Calls its own `customElements.define('tulpar-<name>', ...)` guarded by `if (!customElements.get('tulpar-<name>'))` per the v0.4 micro-frontend safety rule.
- Has a concrete `value` type — no `string | number` unions across the public API.

| Element | `value` type |
|---|---|
| `<tulpar-text-input>` | `string` |
| `<tulpar-textarea>` | `string` |
| `<tulpar-number-input>` | `number \| null` |

## 4. Shared anatomy (FormFieldBase API)

### 4.1 Anatomy ownership

The component **owns its own anatomy**. There is no `<tulpar-form-field>` wrapper in v0.5. Carbon / Vaadin / Mantine / Spectrum pattern. Future wrapper composition can be added non-breakingly later if a real use case emerges.

### 4.2 Label, necessity, and position

```ts
@property({ type: String }) label?: string;
@property({ type: Boolean, reflect: true }) required = false;
@property({ type: String, attribute: 'necessity-indicator' })
necessityIndicator: 'icon' | 'label' | 'none' = 'icon';
@property({ type: String, attribute: 'label-position', reflect: true })
labelPosition: 'top' | 'float' | 'float-in' | 'float-on' | 'none' = 'top';
```

**Necessity indicator:**

- `icon`: required → asterisk after label text; not required → no marker (asymmetric, mark-the-exception).
- `label`: required → "(required)"; not required → "(optional)" (symmetric).
- `none`: no visual marker regardless.

**Label input — attribute vs slot:**

Two input paths:

```html
<!-- Attribute path: simple string -->
<tulpar-text-input label="Email"></tulpar-text-input>

<!-- Slot path: rich content (icon + text, formatting, etc.) -->
<tulpar-text-input>
  <span slot="label">Email <tulpar-icon name="info" /></span>
</tulpar-text-input>
```

If both are set, the **slot wins** (richer content overrides the simpler attribute).

**Label position values:**

| Value | Visual behavior |
|---|---|
| `top` (default) | Label rendered above the input, in a separate row. Classic enterprise default. |
| `float` | Material "over" / notched outline. Label sits inside the input when empty; floats up to sit ON the top border when the input is focused or has value. Label background cuts through the border. |
| `float-in` | Label sits inside the input vertically centered when empty; animates to the inside-top-edge when the input is focused or has value. Stays inside the field — does not escape the border. |
| `float-on` | Label permanently sits ON the top border (notched outline), no animation. Static mid-position version of `float`. |
| `none` | No visible label rendered. The `label` value is still wired to `aria-label` for screen reader users (see A11y below). |

**Default resolution (no `label-position` attribute set):**

- If `label` is set (via attribute or slot) → effectively `top`.
- If `label` is not set → effectively `none` (no empty label row rendered, no wasted vertical space).

Consumers do not need to write `label-position="none"` for label-less fields — the absence of label content is enough.

**Constraints (fallback to `top` + dev-mode warning when violated):**

| Combination | Resolution |
|---|---|
| Any `float*` + `variant="ghost"` | Falls back to `top`. No border to anchor float behavior. |
| Any `float*` + `variant="underlined"` | Falls back to `top`. No top border for notching. |
| `float-in` + `size="xs"` | Falls back to `top`. Compact tier doesn't have vertical room for stacked label + value. |
| `float` or `float-in` + prefix slot has content | Label floats above the **value area only** (not above the prefix). Documented constraint, not a fallback. |

**A11y:**

- `label-position="none"` + `label` value set → component injects `aria-label="<label value>"` on the underlying input. Screen reader announces the field even though no visible label exists. Grid cell editors and toolbar search inputs benefit.
- `label-position="none"` + no `label` value → no `aria-label` injected; consumer is responsible for accessibility (e.g., setting `aria-label` directly on the host).
- All other positions: visible label is linked to the input via `for` / id generation, no `aria-label` needed.

**Truncation in float modes:**

If the label content exceeds approximately 80% of the field width in `float`/`float-in`/`float-on` modes, the floating label truncates visually with `text-overflow: ellipsis`. The underlying `<label>` element retains the full text so screen readers announce the complete label. Consumer-supplied slot label content is the consumer's responsibility — only attribute-driven label gets automatic truncation handling.

**Float label animation:**

- Enter (label floats up): 200ms `ease-out`.
- Exit (label drops back into the field): 150ms `ease-in` (exit faster than enter, per HIG guidance).
- When `prefers-reduced-motion: reduce` is set, the label snaps between positions instantly without animation.

### 4.3 Message row (helper / error / warn)

```ts
@property({ type: String, attribute: 'helper-text' }) helperText?: string;
@property({ type: String, attribute: 'error-text' })  errorText?: string;
@property({ type: String, attribute: 'warn-text' })   warnText?: string;
@property({ type: Boolean, attribute: 'no-message-space' }) noMessageSpace = false;
```

Behavior:

- **Always reserve** a fixed-height row below the input. Default behavior.
- **Opt-out:** `no-message-space` removes the reserved row entirely. For inline/minimal contexts (single search bar in a toolbar, modal with one field).
- Within the row:
  - If `invalid && errorText` → show error text in error color, `role="alert"`.
  - Else if `warn && warnText` → show warn text in warn color, `role="status"`.
  - Else if `helperText` → show helper text in muted color.
  - Else → empty (but space still reserved if `no-message-space` is false).
- Error text **replaces** helper text (does not stack). Single live region per field.

### 4.4 Status taxonomy

Two orthogonal dimensions:

```ts
// Validity (mutually exclusive in display; precedence invalid > warn)
@property({ type: Boolean, reflect: true }) invalid = false;
@property({ type: Boolean, reflect: true }) warn = false;

// Process (orthogonal to validity)
@property({ type: Boolean, reflect: true }) validating = false;
```

This is **a Tulpar differentiator** — 10 of 10 surveyed enterprise systems do not have an async-validation state. The `validating` state is process, not validity. It coexists with `invalid` (e.g., "previous check failed, re-checking now"), `warn`, and the valid state.

Visual mapping:

| State | Border | Status icon |
|---|---|---|
| Default | Neutral | — |
| `invalid && !validating` | Error color | Exclamation |
| `warn && !validating && !invalid` | Warn color | Triangle |
| `validating` | Border keeps prior color | Spinner (overrides exclamation/triangle icon) |

Status icon zone is `display: none` (0 width) when no status is active — no layout shift on toggle.

**Icon swap continuity:** When the active status icon changes (e.g., `validating` becomes true while `invalid` is already true), the icon swap is a 150ms crossfade rather than an instant snap. Maintains spatial continuity per Apple HIG `motion-meaning` guidance. Under `prefers-reduced-motion: reduce`, the swap is instant.

**Validating + screen reader:** When `validating` transitions to `true`, the underlying input gets `aria-busy="true"`, and a polite live region announces "Checking…" (the exact text is localizable via a future i18n surface — v0.5 ships English). Announcement is debounced 150ms to avoid stuttering when `validating` flickers rapidly.

### 4.5 Visual variants

```ts
@property({ type: String, reflect: true })
variant: 'outlined' | 'filled' | 'underlined' | 'ghost' = 'outlined';
```

- `outlined` (default): full bordered box. Form default.
- `filled`: subtle filled background, no border.
- `underlined`: bottom border only, Material-style minimal.
- `ghost`: borderless until focus. Critical for grid cell editor use case (long-term AG Grid roadmap). On focus: outline appears.

Storybook will ship a side-by-side comparison demo since `underlined` and `ghost` are visually adjacent and easy to confuse.

### 4.6 Size scale

```ts
@property({ type: String, reflect: true })
size: 'xs' | 'sm' | 'md' | 'lg' | 'xl' = 'md';
```

5-tier scale. Narrower than Button's 7-tier (no `2xl` / `3xl`) because oversized inputs are visually unsupported in real form layouts. `xs` is the grid-cell-editor tier.

**`xs` constraints (a11y + visual density):**

At `size="xs"` (24px height), auto-managed action buttons (copyable, pastable, clearable, password reveal toggle, NumberInput steppers) cannot meet the 44pt WCAG/HIG touch target minimum and would visually crowd the field. The component **auto-hides** these affordances at xs:

- `copyable`, `pastable`, `clearable`, reveal toggle, steppers: no visual button rendered when `size="xs"`.
- Status icon (auto-managed): still rendered (smaller, ~10px), since it is non-interactive.
- Consumer-supplied `prefix` / `suffix` slot content: still rendered. Consumer accepts responsibility for tap-area sizing on custom slot content.

Rationale: `xs` is for grid cell editors and dense toolbars where action affordances live outside the field (separate buttons in the row), not inside it.

### 4.7 Disabled / readonly

```ts
@property({ type: Boolean, reflect: true }) disabled = false;
@property({ type: Boolean, reflect: true }) readonly = false;
```

Web platform semantics, no Tulpar-specific behavior:

- `disabled`: no interaction, no form submission, `aria-disabled="true"`, focusable false, muted.
- `readonly`: text selectable + copyable, submitted in form, focusable, programmatically settable, no caret edit.

### 4.8 Copyable / pastable

```ts
@property({ type: Boolean, reflect: true }) copyable = false;
@property({ type: Boolean, reflect: true }) pastable = false;
```

When `copyable`: auto-button in auto-suffix area. Click → `navigator.clipboard.writeText(value)`. Icon flips clipboard → checkmark for 1500ms. Live region announces "Copied".

When `pastable`: auto-button. Click → `navigator.clipboard.readText()` → set `value` (after mask filter if applicable). Icon flips clipboard-arrow-down → checkmark. Live region announces "Pasted". Permission denied → X icon 1500ms. Empty clipboard → brief shake (reuses mask-rejected animation).

Both can coexist. Order in auto-suffix lane (TextInput): paste left of copy.

`pastable` is **inert** when `disabled` or `readonly` (setting value contradicts state — button disabled).

A Tulpar differentiator — no surveyed enterprise system ships built-in copyable/pastable affordances.

### 4.9 Prefix / suffix slot model

Two named slots: `prefix` and `suffix`. Three-zone layout inside the field:

```
[prefix slot] [input value] [auto-suffix area] [suffix slot]
   user          native          system            user
```

Per-component, the auto-suffix area contains different built-ins (status icon, password reveal, paste/copy, clear X, steppers).

**Pointer events:**

- Default: slot host CSS sets `pointer-events: none`. Clicks on prefix/suffix content fall through to focus the input.
- Opt-in interactive: boolean attributes `prefix-interactive` and `suffix-interactive` flip the slot host to `pointer-events: auto`. Mantine's `leftSectionPointerEvents` simplified to a boolean.

**Content guidance** (documented, not enforced):

- Slot content should be a single icon or a short text label (max ~3ch).
- For adjacent buttons (search input + "Send" button), compose at the parent level using a stack/group layout — out of scope for the input itself.

### 4.10 Width

`:host { display: block; width: 100% }` by default. Consumer constrains via CSS (`style="width: 200px"` or flex parent). No `width` attribute on the component — `style` is the standard Web Platform mechanism.

## 5. TextInput

`<tulpar-text-input>` — single-line, covers `text | email | url | tel | search | password` via `type`.

### 5.1 Public API

```ts
// Inherited from FormFieldBase: label, required, necessityIndicator,
// helperText, errorText, warnText, invalid, warn, validating,
// disabled, readonly, size, variant, copyable, pastable,
// noMessageSpace, prefix slot, suffix slot.

@property({ type: String, reflect: true })
type: 'text' | 'email' | 'url' | 'tel' | 'search' | 'password' = 'text';

@property({ type: String }) value = '';
@property({ type: String }) placeholder?: string;
@property({ type: String, reflect: true }) name?: string;
@property({ type: String }) autocomplete?: string;
@property({ type: Number, attribute: 'maxlength' }) maxLength?: number;
@property({ type: Number, attribute: 'minlength' }) minLength?: number;
@property({ type: String }) pattern?: string;
@property({ type: Boolean, reflect: true }) clearable = false;
@property({ type: Boolean, attribute: 'show-count' }) showCount = false;
@property({ type: Boolean, attribute: 'no-reveal-toggle' }) noRevealToggle = false;

// Mask (see section 7)
@property({ type: String }) mask?: string;
@property({ type: String, attribute: 'mask-emit' }) maskEmit: 'masked' | 'raw' = 'masked';
@property({ type: String, attribute: 'mask-display' }) maskDisplay: 'eager' | 'lazy' = 'eager';
@property({ type: String, attribute: 'mask-slot-char' }) maskSlotChar = '_';

// Read-only computed property
get rawValue(): string;
```

### 5.2 Type-driven auto-enrichment

The "auto-enrichment" pattern is consistent with Tulpar's broader baked-in-differentiators philosophy (copyable, pastable, validating, mask).

**`type="password"`:**
- Auto reveal toggle button in auto-suffix area. Eye icon.
- Click: toggles underlying `<input>` between `type="password"` and `type="text"`.
- `aria-label` dynamic: "Show password" ↔ "Hide password".
- Opt-out: `no-reveal-toggle` boolean.

**`type="search"`:**
- Auto search-icon prefix when the `prefix` slot is empty (slot fallback). User-provided prefix content wins.
- `clearable=true` is implicit. Explicit `clearable="false"` opt-out.
- Native browser search behavior (mobile keyboard "search" return key) inherited from underlying `<input type="search">`.

### 5.3 Autocomplete dev warning

When `type ∈ {email, password, tel}` and `autocomplete` is unset, log a single `warnDev` (per `_internal/warn-dev.ts`) in development. Production: no-op. Mirrors Polaris's lint-level discipline without enforcing it.

### 5.4 Clearable button

When `clearable=true` and `value !== ''`:

- Auto button (X icon) in auto-suffix area.
- Click → `value = ''`, focus returns to input, dispatches `change` event.
- Inert when `disabled` or `readonly`.

### 5.5 Char counter

When `show-count=true`:

- Renders in the message row, right-aligned.
- Format: `<currentLength>` (no maxlength) or `<currentLength> / <maxLength>` (maxlength set).
- Counter switches to error color when length === maxLength (visual cue independent of `invalid`).

### 5.6 Auto-suffix lane ordering (TextInput)

Inside the field, left to right after the value:

```
[input value] → [status icon] → [reveal toggle if pwd] → [paste] → [copy] → [clear X] → [user suffix slot]
```

Each element is rendered only when its condition applies. The user suffix slot is always rightmost (AntD/Mantine convention).

## 6. Textarea

`<tulpar-textarea>` — multi-line text.

### 6.1 Public API

```ts
// Inherited from FormFieldBase, minus prefix/suffix slot (not exposed on Textarea).

@property({ type: String }) value = '';
@property({ type: String }) placeholder?: string;
@property({ type: String, reflect: true }) name?: string;
@property({ type: Number, attribute: 'maxlength' }) maxLength?: number;
@property({ type: Number, attribute: 'minlength' }) minLength?: number;
@property({ type: Boolean, attribute: 'show-count' }) showCount = false;

// Autosize
@property({ type: Boolean }) autosize = true;
@property({ type: Number, attribute: 'min-rows' }) minRows = 2;
@property({ type: Number, attribute: 'max-rows' }) maxRows = 6;
@property({ type: Number }) rows?: number;  // when set, disables autosize

// Resize handle (CSS resize property)
@property({ type: String, reflect: true })
resize: 'none' | 'both' | 'horizontal' | 'vertical' = 'vertical';
```

### 6.2 Autosize

Default `autosize=true`, `min-rows=2`, `max-rows=6`. Content-driven height between bounds. Implemented via `scrollHeight` measurement after each input event (debounced via Lit `updated()`).

Setting `rows` to any number disables autosize and uses fixed height.

### 6.3 Resize handle

Default `resize="vertical"` — native browser handle in bottom-right corner. Atlaskit-style attribute mapping: directly assigned to CSS `resize` property on the underlying `<textarea>`.

### 6.4 No prefix/suffix slots

Textarea does **not** render `prefix` / `suffix` slot containers. Multi-line + prefix/suffix is visually awkward and not supported by AntD or Mantine. The FormFieldBase declares the slots in the abstract, but Textarea's render method omits the slot markup entirely. Other auto-managed affordances (status icon, copyable, pastable) still render in the top-right corner area of the textarea.

### 6.5 Char counter — overlay placement

When `show-count=true`, the counter renders **inside** the textarea, bottom-right corner, semi-transparent overlay (AntD pattern). The textarea's `padding-bottom` increases to accommodate the counter — content does not flow under it. This is intentional divergence from TextInput's message-row placement: a long textarea would push the message row off-screen, hiding the counter from the user.

### 6.6 Auto-suffix area (Textarea)

Positioned top-right corner of the textarea field:

```
[status icon] → [paste?] → [copy?]
```

`clearable` is not exposed on Textarea (rare UX; user can select-all + delete). Steppers are obviously not applicable.

## 7. TextInput mask

The mask system is **TextInput-only** — Textarea is multi-line (mask doesn't apply), NumberInput has its own `Intl.NumberFormat` formatter.

This is a Tulpar differentiator: 8 of 10 surveyed enterprise systems do not ship built-in masking. PrimeNG ships a dedicated `<InputMask>` component; Tulpar's approach is integrated into TextInput (no extra import).

### 7.1 Token alphabet

| Token | Meaning | Behavior |
|---|---|---|
| `A` | UPPERCASE letter required | Lowercase input is auto-uppercased |
| `a` | lowercase letter required | Uppercase input is auto-lowercased |
| `L` | Letter required, case preserved | Both cases accepted as-is |
| `9` | Digit `0`–`9` required | Non-digit rejected |
| `S` | Special character required | Printable non-alphanumeric: ``!@#$%^&*()-_=+[]{};:'",.<>?/\|`~`` |
| `*` | Any printable character | Letter, digit, special — all accepted |
| `\X` | Literal `X` | Escape next char as literal (so `\9` literally means "9", not the digit token) |
| Anything else | Literal | Spaces, `-`, `/`, `(`, `)`, `+`, etc. auto-treated as literals |

### 7.2 Examples

| Use case | Mask string |
|---|---|
| TR mobile phone | `+90 (999) 999 99 99` |
| TR license plate | `99 AAA 999` |
| Credit card | `9999 9999 9999 9999` |
| ISO date | `9999-99-99` |
| TR date | `99/99/9999` |
| Username (lowercase fixed) | `aaaaaaaa` |
| Strong password fragment | `AaAa99SS` |

### 7.3 Value semantics

```ts
@property({ type: String, attribute: 'mask-emit' })
maskEmit: 'masked' | 'raw' = 'masked';
```

- `mask-emit="masked"` (default): form / `value` property emits the full displayed string with literals. E.g., `"+90 (532) 123 45 67"`.
- `mask-emit="raw"`: form / `value` property emits only token-matched characters. E.g., `"5321234567"`.

**`rawValue` DOM property is always available** regardless of `mask-emit`. Read-only computed getter. Bonus affordance — no API conflict, never costs anything.

### 7.4 Display style

```ts
@property({ type: String, attribute: 'mask-display' })
maskDisplay: 'eager' | 'lazy' = 'eager';
```

- `eager` (default): empty token slots are shown with the slot character (default `_`), full template visible (`+90 (___) ___ __ __`).
- `lazy`: only literals are visible when empty (`+90 (   )    `), tokens appear as the user types.

Slot character override: `mask-slot-char="·"` (or any single character).

### 7.5 Invalid char behavior

Silent reject + `data-mask-rejected` attribute set on the host element for 200ms. Consumer CSS can hook this attribute for a shake animation:

```css
tulpar-text-input[data-mask-rejected] {
  animation: tulpar-mask-shake 200ms;
}
```

The same `data-mask-rejected` attribute drives the empty-clipboard paste shake feedback (section 4.8).

### 7.6 Paste behavior

Pasted text is auto-cleansed: token-matching characters are extracted in order from the pasted string and fill remaining token slots left-to-right. Literals in the mask are not consumed from the paste.

Example: mask `+90 (999) 999 99 99`, paste `"905321234567"` → fills as `+90 (532) 123 45 67`.

### 7.7 Optional tokens — explicitly out of scope

No `?` modifier or `[optional]` blocks in v0.5. All masks are fixed-length / fully-required. Adding optional tokens later is not a breaking change.

### 7.8 Implementation note

Mask logic lives in `_internal/mask-engine.ts` as a Lit `ReactiveController`. TextInput opts into it when `mask` attribute is set. Engine handles:

- Token parsing (compile mask string into a token array on `mask` attribute change).
- Input filtering on `beforeinput` event.
- Bidirectional value transforms (`masked ↔ raw`).
- Cursor position management (when typing into the middle of a mask, cursor advances over literals).
- Paste handler.

## 8. NumberInput

`<tulpar-number-input>` — numeric input with `Intl.NumberFormat` locale formatting and steppers.

### 8.1 Public API

```ts
// Inherited from FormFieldBase: label, required, necessityIndicator,
// helperText, errorText, warnText, invalid, warn, validating,
// disabled, readonly, size, variant, copyable, pastable,
// noMessageSpace, prefix slot, suffix slot.

@property({ type: Number }) value: number | null = null;
@property({ type: String, reflect: true }) name?: string;
@property({ type: Number }) min?: number;
@property({ type: Number }) max?: number;
@property({ type: Number }) step = 1;
@property({ type: String }) placeholder?: string;
@property({ type: Boolean, attribute: 'allow-empty' }) allowEmpty = true;
@property({ type: Boolean, attribute: 'hide-steppers', reflect: true }) hideSteppers = false;
@property({ type: Boolean, attribute: 'integer-only', reflect: true }) integerOnly = false;

// Format — shorthand attribute layer (HTML-friendly)
@property({ type: String, attribute: 'format-style' })
formatStyle: 'decimal' | 'currency' | 'percent' = 'decimal';
@property({ type: String }) currency?: string;                 // ISO 4217 (TRY, USD, ...)
@property({ type: String }) locale = navigator.language;        // BCP47
@property({ type: Number, attribute: 'min-fraction-digits' }) minFractionDigits?: number;
@property({ type: Number, attribute: 'max-fraction-digits' }) maxFractionDigits?: number;
@property({ type: Boolean, attribute: 'use-grouping' }) useGrouping = true;
@property({ type: String, attribute: 'format-prefix' }) formatPrefix?: string;
@property({ type: String, attribute: 'format-suffix' }) formatSuffix?: string;

// Format — full Intl property (advanced)
@property({ attribute: false }) formatOptions?: Intl.NumberFormatOptions;

// Long-press acceleration
@property({ type: Number, attribute: 'step-hold-delay' }) stepHoldDelay = 500;
@property({ type: Number, attribute: 'step-hold-interval' }) stepHoldInterval = 50;
```

### 8.2 Locale formatting (hybrid API)

Two layers:

1. **Shorthand attributes** (HTML-friendly, ~9 attributes): `format-style`, `currency`, `locale`, `min/max-fraction-digits`, `use-grouping`, `format-prefix`, `format-suffix`. Covers ~90% of real use cases.
2. **`.formatOptions` property** (JS-only): full `Intl.NumberFormatOptions`. Escape hatch for `notation`, `compactDisplay`, `currencySign`, `signDisplay`, `roundingMode`, `unit`, etc.

**Precedence:** if both are set, `.formatOptions` wins over shorthand attributes. The component internally builds an `Intl.NumberFormat` from `{ ...shorthandDerivedOptions, ...formatOptions }`.

**Examples:**

```html
<!-- Currency, TR locale -->
<tulpar-number-input
  format-style="currency"
  currency="TRY"
  locale="tr-TR"
  min-fraction-digits="2"
  max-fraction-digits="2"
  min="0"
></tulpar-number-input>
<!-- Display: ₺1.234,56 -->

<!-- Percent -->
<tulpar-number-input
  format-style="percent"
  locale="tr-TR"
  max-fraction-digits="1"
  min="0"
  max="100"
></tulpar-number-input>
<!-- Display: %12,5 -->

<!-- Integer count with unit label -->
<tulpar-number-input
  integer-only
  min="0"
  max="99999"
  use-grouping
  format-suffix=" adet"
></tulpar-number-input>
<!-- Display: 1.234 adet -->
```

```ts
// Advanced: compact accounting notation via property
const el = document.querySelector('tulpar-number-input');
el.formatOptions = {
  style: 'currency',
  currency: 'TRY',
  currencySign: 'accounting',  // (₺1.234,56) for negatives
  notation: 'compact',          // ₺1,2 Mn for large values
  compactDisplay: 'short',
};
```

### 8.3 Steppers

- Visible by default, vertically stacked at the right edge of the field. Top = increment, bottom = decrement.
- Opt-out: `hide-steppers` boolean.
- Long-press acceleration: holding a stepper button starts repeating after `step-hold-delay` ms (default 500ms), then continues every `step-hold-interval` ms (default 50ms). Mantine pattern.
- Disabled when value is at the bound (`value === max` disables increment; `value === min` disables decrement).
- Disabled entirely when `disabled` or `readonly`.

### 8.4 Clamp behavior

**Blur-time clamping** (Spectrum pattern):

- User can type any number, including out-of-range, during input.
- On blur (or Enter / form submission), value is clamped to `[min, max]`.
- During typing, `invalid` flag may be set to surface out-of-range as a status (consumer chooses whether to bind it).

No `clamp-behavior` knob in v0.5. If a real "strict on type" use case surfaces later, it can be added non-breakingly.

### 8.5 Keyboard

Native input handles arrow keys. Tulpar layers Spectrum modifier conventions:

- `ArrowUp` / `ArrowDown` → `±step`
- `Shift + ArrowUp / Down` → `±step * 10`
- `Ctrl/Cmd + ArrowUp / Down` → `±step * 100`
- `Home` → set to `min` (if defined)
- `End` → set to `max` (if defined)

### 8.6 Mouse wheel

**Disabled by default.** 10 of 10 surveyed enterprise systems disable wheel-scroll-to-increment as an a11y / accidental-edit hazard. No opt-in attribute in v0.5 — if a real use case appears, add later.

### 8.7 Integer-only mode

`integer-only` boolean is a shorthand. When true, the component internally enforces:

- `step` defaults to `1` if not explicitly set.
- `max-fraction-digits=0` (overrides format options).
- Decimal separator key input rejected with the `data-mask-rejected` 200ms attribute (reuses mask animation hook).

### 8.8 Allow empty

Default `allow-empty=true`. Empty input → `value = null`. The component does not snap to `min` on empty unless `allow-empty=false`.

### 8.9 Format prefix / suffix vs slot prefix / suffix

NumberInput exposes both:

- `format-prefix=""` / `format-suffix=""` — **string literals glued to the formatted number**, inside the field, no slot. E.g., `format-suffix=" adet"`.
- `prefix` / `suffix` slots (inherited from FormFieldBase) — **arbitrary content**, e.g., an icon or button.

For currency, prefer the Intl-native path (`format-style="currency" currency="TRY"`) — only use `format-prefix` for non-currency labels that Intl doesn't model (e.g., custom unit "TL" instead of "₺").

### 8.10 Auto-suffix lane ordering (NumberInput)

```
[slot prefix] [format-prefix] [input value] [format-suffix] [status icon] [paste?] [copy?] [steppers] [slot suffix]
```

Steppers occupy a stable position to the left of the slot suffix. `clearable` is **not** supported on NumberInput — `allow-empty=true` already lets Delete/Backspace clear the value, and a clear button would compete visually with steppers.

## 10. Accessibility

- `aria-required="true"` when `required`.
- `aria-invalid="true"` when `invalid`.
- `aria-busy="true"` on the underlying input when `validating=true`.
- `aria-describedby` auto-generated, points to message row content (helper / error / warn).
- Error region has `role="alert"` when active (live announcement on appearance).
- Validating live region has `role="status"` + `aria-live="polite"` with brief "Checking…" announcement when `validating` transitions to true (debounced 150ms to absorb rapid flicker).
- Asterisk required marker has `aria-hidden="true"` — `aria-required` carries the semantics.
- Auto-suffix buttons each have a dynamic `aria-label`: "Copy value", "Paste value", "Clear value", "Show password" / "Hide password", "Increment", "Decrement".
- Status icon zone is `aria-hidden="true"` — text-based status (errorText / warnText) carries the meaning.
- Steppers reachable via Tab (after the input), or `aria-controls` association with the input for screen readers.

**Color contrast:**

- All foreground/background pairs (text, icons, focus rings, borders) meet **WCAG AA 4.5:1** at minimum. Targeted pairs (label, helper text, primary value text) target **AA+ 7:1** for safety margin across light and dark modes.
- Specifically, helper-text token (`--tulpar-input-message-helper`) maps to a higher-contrast neutral (e.g., `c.stone[600]` on light surface = 7.49:1, not `c.stone[500]` which is marginal at 4.62:1).
- Focus ring contrast verified against ghost variant's transparent background (focus ring must remain visible without a surrounding border to lean on).

**Reduced motion:**

All animations respect `@media (prefers-reduced-motion: reduce)`:

- Float label animation: disabled (snap between positions).
- Status icon swap crossfade: disabled (instant swap).
- Validating spinner: replaced with a non-animated alternative indicator (static "⋯" or pulsing opacity step rather than continuous rotation).
- Mask-rejected shake (200ms): disabled (instant state change).
- Copy/paste confirmation icon flip (1500ms hold): the visual confirmation icon still appears, but without slide/scale transition.
- Stepper long-press acceleration: unaffected (this is a user-initiated repeat, not a decorative animation).

## 11. Form association

All three components are form-associated custom elements (`static formAssociated = true`, `attachInternals()`), continuing the v0.4 Button pattern.

- `name` / `value` reflection.
- `setFormValue(value)` called on every value change.
- `formResetCallback()` → reset to initial value.
- `formDisabledCallback()` → propagate fieldset disabled state.
- No `ControlValueAccessor` in the Angular wrapper — native form-association bubbles to Reactive Forms automatically.

## 13. Testing strategy

### 13.1 Core (Web Test Runner + open-wc/testing)

For each component, test groups:

1. **Rendering & defaults** — host element, slot anatomy, default attributes.
2. **Value contract** — get/set, reflection, FormData submission, FACE compliance.
3. **Status states** — invalid / warn / validating combinations, ARIA wiring, icon presence.
4. **Anatomy** — label rendering (attribute + slot paths, slot wins), label-position values + fallback behavior under unsupported variant/size combinations, default resolution (label-set → `top`, label-unset → `none`), `aria-label` wiring in `none` mode, message row reservation, `no-message-space` opt-out, helper/error/warn text replacement rules.

6. **Keyboard** — input behavior, Enter for form submit, modifier keys for NumberInput.

   - TextInput: `type` variants behavior, password reveal toggle, search auto-enrichment, char counter, autocomplete dev warning.
   - TextInput mask: token compilation, eager/lazy display, value emit modes, rawValue property, invalid-char reject + `data-mask-rejected`, paste cleansing, cursor movement.
   - Textarea: autosize bounds, resize attribute → CSS, char counter overlay placement.
   - NumberInput: Intl formatter coverage (decimal/currency/percent/integer-only/format-prefix-suffix), `.formatOptions` precedence, clamp on blur, steppers + long-press acceleration, integer-only decimal rejection, allow-empty.

## 15. Release & sequencing

### 15.1 Branch model

Long-lived `release/v0.5-input-family`. Each component on its own topic branch:

- `feat/v0.5-form-field-base`
- `feat/v0.5-text-input` (depends on form-field-base)
- `feat/v0.5-text-input-mask` (depends on text-input)
- `feat/v0.5-textarea` (depends on form-field-base)
- `feat/v0.5-number-input` (depends on form-field-base)

Each merges into release branch. Release branch → `main` as a single minor bump (v0.5.0).

### 15.2 Build order (recommended)

1. **FormFieldBase + tokens.** Skeleton, no public custom-element definition. Build out anatomy, message row, status visuals.
2. **TextInput** (no mask). Validates FormFieldBase end-to-end.
3. **TextInput mask engine.** Iterative — tokens first, then display modes, then paste + cursor logic.
4. **Textarea.** Diverges from FormFieldBase (no prefix/suffix slot, char counter overlay).
5. **NumberInput.** Largest specific surface — Intl formatter, steppers, clamp.

### 15.3 Changesets

One `minor` changeset listing the 3 new components + FormFieldBase plumbing.

### 15.4 Migration guide

`docs/migration/v0.4-to-v0.5.md` — additive, no breaking changes. Lists new components, example usage per common scenario (signin form, settings form, currency entry, grid cell editor).
