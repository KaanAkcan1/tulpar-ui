# Tulpar UI v0.14 — Select (single)

**Status:** Draft — pending spec-review loop + user approval
**Date:** 2026-06-23
**Predecessor:** v0.13 Display & Status Atoms (reuses `<tulpar-spinner>` + `_internal/tone/`)
**Type:** New form-control element + package-private `_internal/listbox/` module + tokens touch + Angular/Vue wrappers
**Roadmap:** First slice of **Wave 2 — Overlay-anchored data entry & menus** (see [`docs/roadmap.md`](../../roadmap.md))

---

## 1. Overview

A single-select dropdown form control: `<tulpar-select>` with slotted `<tulpar-option>` /
`<tulpar-option-group>` children. It is the most-demanded missing input and the first Wave 2 component;
it establishes the **shared listbox internal** (collection + active-index + typeahead +
`aria-activedescendant` state machine) that Combobox / MultiSelect / Menu will later reuse.

It subclasses the existing `FormFieldBase`, so it inherits the entire input "field" anatomy (label +
positions, helper/error/warn message row, status icons, required marker, sizes `xs–xl`, variants
`outlined/filled/underlined/ghost`, native form-association) for free — the Select only implements its
own `renderControl()` (the combobox trigger) plus the portaled listbox.

### Goals
- A correct, accessible single-select that is **pixel-aligned with the text-input family** in a form grid.
- The **WAI-ARIA APG "Select-Only Combobox"** pattern done right: `combobox`+`listbox`,
  `aria-activedescendant` virtual focus (focus pinned to the trigger), typeahead, full keyboard map.
- A listbox that **never clips** — even inside scroll containers / transformed ancestors (the future
  data grid) — via the native **Popover API top-layer**, falling back to `position: fixed`.
- Rich options: optional leading icon + two-line description, per-option `disabled`, grouped headers.
- Three **independently legible** option signals (selected / active / hover) that stack without confusion.
- Establish `_internal/listbox/` as the reusable Wave-2 foundation (one consumer now, designed for reuse).
- Angular + Vue wrappers and **playground parity** (ng + vue), Storybook stories, WTR + Playwright verify.

### Non-Goals (binding) — see §11
`options` **data-prop** (array form) · **Combobox** (free-text filtering) · **MultiSelect** ·
**virtualization** (thousands of options) · **in-popup search field** · **mobile-native `<select>`
fallback**. Some leave **reserved API surface** (noted inline) so they are additive later, not breaking.

### Cross-cutting principle: prop AND slot, never one instead of the other
Per [[feedback_optional_prop_does_not_remove_slot]], every content affordance has both a convenience prop
and a slot: option `label` (prop) + default slot; `description` (prop) + slot; group `label` (prop) + slot.
The slot wins when both are set.

### 2.1 Listbox rendering strategy — shadow-contained + Popover-API top-layer (the load-bearing decision)

The hard constraint: **slotted `<tulpar-option>` light-DOM children are only projected by a `<slot>` that
lives in the Select's own shadow DOM.** A `<slot>` relocated into the shared overlay-root portal projects
nothing — so the v0.11 "relocate content into the portal root" model (tooltip/popover, separate `for`-id
nodes) **does not apply** to Select, which owns trigger + listbox in one element.

Decision (confirmed): **keep the listbox inside the Select's shadow DOM and lift it to the top layer with
the native Popover API** (`popover="manual"`), positioned by the existing `positioner` (offset → flip →
shift). The top layer escapes ALL clipping — `overflow`, transformed/`filter`/`contain` ancestors — which
`position: fixed` alone does not guarantee (real risk in the future grid). Where the Popover API is
unsupported, fall back to `position: fixed`. The element registers on the existing **overlay-root Escape
stack** (so an in-shell/in-dialog Select closes before its parent overlay) — stack only, no content move.

- Capability probe `supportsPopover()` already exists in [`_internal/overlay/anchor.ts`](../../packages/core/src/_internal/overlay/anchor.ts).
- The positioner is pure rect math ([`_internal/overlay/positioner.ts`](../../packages/core/src/_internal/overlay/positioner.ts)) — the Select measures trigger/listbox/viewport rects and applies the result; no engine change.
- **Why not relocate (eliminated):** moving the slotted nodes breaks slot assignment; re-rendering options
  as a data model in a body portal is the documented WC trap (confirmed across Shoelace/Material/Carbon).

### 2.2 The `_internal/listbox/` module (the reusable Wave-2 foundation)

Three small, DOM-light, independently unit-tested units. The Select orchestrates; options are "dumb"
display elements that receive `data-active`/`data-selected`/`aria-selected` from the Select.

- **`collection.ts`** — reads slotted `<tulpar-option>` (and `<tulpar-option-group>`) children into an
  ordered model `{ value, label, disabled, index, el }`, recomputed on `slotchange`. Group membership is
  recorded for ARIA wiring. Label is derived from the `label` prop or trimmed slot text (ignore comment
  nodes — the [[reference_lit_slotchange_requestupdate_loop]] / empty-slot lesson applies).
- **`active-index.ts`** — pure active-index reducer: `next`/`prev` over **enabled** options only (skip
  disabled), `first`/`last`, **no-wrap** (stop at ends), and "set active = selected (or first enabled)"
  on open.
- **`typeahead.ts`** — buffered matcher: accumulate typed chars with a **500ms** reset, `startsWith`
  (case-insensitive, locale-aware), same-letter repeat cycles through matches.

Per the `_internal` rule, this stays package-private (never exported, no cross-package import). One
consumer (Select) now; the API is shaped so Combobox/MultiSelect/Menu adopt it without a rewrite. No
premature extraction to a separate package (that waits for a real second consumer per
[`docs/architecture/internal-layering.md`](../../architecture/internal-layering.md)).

### 2.3 Unit boundaries
`tulpar-option` / `tulpar-option-group` hold no selection logic — they render content and reflect state
attributes set by the parent. All orchestration lives in `tulpar-select` + the three `_internal/listbox/`
units. Each listbox unit is pure and testable without a DOM tree. The Select is understandable without
reading option internals, and vice-versa.

## 3. Public API

### 3.1 `<tulpar-select>` (extends `FormFieldBase`)

**Inherited from `FormFieldBase`** (no new work): `name · size (xs|sm|md|lg|xl) · variant
(outlined|filled|underlined|ghost) · disabled · readonly · label · label-position
(top|float|float-in|float-on|none) · required · necessity-indicator · helper-text · error-text ·
warn-text · invalid · warn · validating` + `prefix`/`suffix` slots + the message row + status zone.

**New Select-specific members:**

| Member | Type | Default | Notes |
|---|---|---|---|
| `value` | string (attr+prop) | `""` | Selected option's `value`. `""` = no selection. Reflected for styling. |
| `placeholder` | string | `""` | Muted trigger text shown ONLY when no value. **Presentation-only — never an option, never the accessible name** (§4.4). |
| `clearable` | boolean | `false` | Show a ✕ to return to no-selection. Auto-suppressed when `required` or when empty (§4.4). |
| `loading` | boolean (reflect) | `false` | Trigger shows a spinner in place of the chevron; listbox shows the loading row (§5.5). |
| `open` | boolean (reflect) | `false` | Open state. Reflected (`[open]`/`data-open`) for CSS and programmatic open/close. |

**`readonly` semantics:** focusable, shows the value, but cannot open (distinct from `disabled`, which is
non-focusable + muted). Reuses FormFieldBase's readonly visual.

**Events:**
- **`change`** — `CustomEvent<{ value: string }>`, `bubbles`, fired on **commit** (option chosen, or
  cleared). Mirrors native `<select>` semantics. (No separate `input` event for v1 — there is no
  intermediate typing; the active highlight is not a value change.)
- Form value is set via `ElementInternals.setFormValue(value)` on every commit, so Angular Reactive Forms
  and Vue native binding work with **no extra ControlValueAccessor** (matches the input family).
- *(Reserved, deferred:)* `tulpar-open` / `tulpar-close` are NOT emitted in v1; reserved names if needed.

**Public methods:** `focus()` lands DOM focus on the trigger (used by "focus first invalid on submit").

### 3.2 `<tulpar-option>` (dumb display element)

| Member | Type | Default | Notes |
|---|---|---|---|
| `value` | string (**required**) | — | Becomes the Select's `value` on commit. Dev-warn if missing/duplicate. |
| `label` (prop) **or** default slot | string / slotted | — | Display text. **Dual**: slot wins. Used for typeahead + trigger display. |
| `description` (prop) **or** `description` slot | string / slotted | — | Optional second line. |
| `icon` slot | slotted | — | Optional leading icon (adopts brand color when selected). |
| `disabled` | boolean | `false` | Not selectable; skipped by keyboard nav + typeahead; pointer click is a no-op. |

Reflected (set by parent, read-only to consumers): `aria-selected`, `data-selected`, `data-active`, a
stable `id` (for `aria-activedescendant`), `role="option"`.

### 3.3 `<tulpar-option-group>`

| Member | Type | Notes |
|---|---|---|
| `label` (prop) **or** `label` slot | string / slotted | Group header text. Renders a non-interactive sticky header (§5.4). |

Contains `<tulpar-option>` children. Gets `role="group"` + `aria-labelledby` → the header id. Visual
separation between groups is spacing (no separate divider element in v1; see §11 reserved).

### 3.4 Trigger accessible name & description
The trigger's accessible name is `aria-labelledby` → the FormFieldBase label (NEVER the placeholder).
Helper/error are wired via `aria-describedby` reusing FormFieldBase's `_ariaDescribedBy()`/`_msgId`.

## 4. Interaction & accessibility

### 4.1 ARIA pattern (WAI-ARIA APG "Select-Only Combobox")
- **Trigger:** a focusable element with `role="combobox"` (NOT a native `<button>` — `button` role can
  suppress some SR list semantics), `aria-haspopup="listbox"`, `aria-expanded`, `aria-controls=<listbox
  id>`, `aria-labelledby`, `aria-describedby`, `aria-activedescendant=<active option id | unset>`,
  `aria-invalid` mirrored from the field state.
- **Listbox:** `role="listbox"`; options `role="option"` + `aria-selected`; groups `role="group"` +
  `aria-labelledby`.
- **Virtual focus:** DOM focus **stays on the trigger** the whole time the listbox is open. The active
  option is tracked by `aria-activedescendant` + a `data-active` style — **never** roving real DOM focus
  into options (the disputed Radix pattern; rejected). This is what keeps typeahead, Escape, and focus
  restore correct.

### 4.2 Keyboard map
**Closed (focus on trigger):** `Enter` / `Space` / `Alt+ArrowDown` / `ArrowDown` / `ArrowUp` → open (set
active = selected, else first enabled; scroll it into view). A printable char → open + typeahead-jump.

**Open:**
- `ArrowDown` / `ArrowUp` → move active to next/prev **enabled** option; **no-wrap** (stop at ends).
- `Home` / `End` → first / last enabled.
- `PageDown` / `PageUp` → move by ~visible-page (nice-to-have).
- printable chars → typeahead (500ms buffer, `startsWith`, same-letter cycling).
- `Enter` → commit active option, close, focus → trigger.
- `Space` → commit (no text entry, so Space selects, mirroring native).
- `Tab` / `Shift+Tab` → **commit** active option, then move focus (native `<select>` behavior).
- `Escape` → cancel: close **without changing value**, focus → trigger, `stopPropagation`.
- `Alt+ArrowUp` → close, keep selection.

On **every** active change and on open: `scrollIntoView({ block: 'nearest' })` the active option.

### 4.3 Open / close / dismissal & focus
- **Open:** click/tap trigger or the open keys. Click on an open trigger toggles closed.
- **Dismissal & value:**
  - select option (click/Enter/Space/Tab) → **commit**, close, focus → trigger.
  - outside pointerdown → close, **keep value** (no commit of the merely-active option). "Outside" is
    computed against the **union of trigger + portaled panel** (the portaled-popover trap), via a
    `pointerdown` listener.
  - `Escape` → close, **revert** to pre-open value, focus → trigger, `stopPropagation`.
  - ancestor/page **scroll** or window **resize** → close (don't glue a detached panel through scroll);
    focus → trigger.
- **Focus restore is non-negotiable:** every close path returns DOM focus to the trigger — losing focus to
  `<body>` is the most common portaled-popup a11y regression. Verified in playgrounds.
- Hover model: **pointer-enter moves `aria-activedescendant`** to the hovered option → hover and active are
  one unified highlight (mouse and keyboard never show two different cursors).

### 4.4 Placeholder vs label vs empty value; clearable
- **Label** (FormFieldBase) is the accessible name and always present. **Placeholder** is a
  presentation-only muted string shown only in the no-selection state; it is **never** an option in the
  list and **never** the accessible name. **Empty / no-selection** is a first-class state distinct from
  "an option whose label is empty."
- **Clearable:** opt-in (`clearable`), shown only when a value is selected; **auto-suppressed when
  `required`** (clearing would just create an error) or when empty. The ✕ is a separate focusable control
  (`aria-label="Clear selection"`), stops propagation (does not open the listbox), keeps focus on the
  trigger, sets `value=""`, and fires `change`.
- Required is indicated on the label (asterisk + necessity text), not the placeholder.

### 4.5 Validation timing & wiring
- Validate at **commit/close (blur-equivalent)** and at **form submit** — never continuously, and never on
  a pristine untouched field. A required-empty Select goes invalid when opened-and-closed-without-choosing
  or on submit.
- Visual invalid/warn reuses FormFieldBase: danger/warn border + status icon + message-row text
  (color-never-alone). `aria-invalid="true"` on the trigger; error id in `aria-describedby`.
- Message-row live region reuses FormFieldBase's existing plumbing — polite for blur/inline errors,
  escalate to `role="alert"` for submit-time. Do **not** invent a parallel message row.

### 4.6 Listbox content states
- **Empty:** non-interactive centered "No options" row (overridable via `empty` slot/prop). Trigger still
  opens to show it; the panel is dismissable.
- **Loading** (`loading`): trigger spinner + a listbox "Loading…" row reusing **`<tulpar-spinner>`** (from
  v0.13). Keep panel min-height stable so results don't jump. `aria-busy` while loading.
- **Error:** centered danger row "Couldn't load options" (overridable) + optional focusable "Retry". These
  are **listbox-content** states — the trigger stays neutral (not field-validation; don't redden it).

## 5. Visual design

Direction: **restrained-precise** (Linear / Stripe / Vercel register). The trigger is visually identical to
a TextInput of the same variant/size (so a mixed form reads as one grid); the listbox is a lighter, flatter
sibling of the popover — a menu, not a card. Token discipline: **semantic tokens only**, reusing the entire
`--tulpar-input-*` family (trigger) and `--tulpar-overlay-*` family (listbox); every `var()` carries a
light fallback. No primitive tokens in component CSS (ESLint-enforced).

### 5.1 Trigger
Renders through FormFieldBase's control row (same label positions, message row, status zone, sizes). It is
a `role="combobox"` element that **must equal a same-variant TextInput pixel-for-pixel**.
- Layout: `inline-flex; align-items:center; gap:.5rem` — leading slot (selected option's icon, optional) ·
  value/placeholder (flex-grows, `overflow:hidden; text-overflow:ellipsis; white-space:nowrap; text-align:start`)
  · trailing cluster (`margin-inline-start:auto`): optional ✕, then the chevron.
- Value text `--tulpar-input-text-default`, weight ~450; placeholder `--tulpar-input-text-placeholder`
  (no italic). No layout shift placeholder→value (same flex slot). Float/float-in label suppresses the
  placeholder exactly as inputs do.
- **Variants** inherited verbatim from `form-field-base.styles.ts` (outlined border / filled subtle bg /
  underlined bottom-rule only / ghost transparent-until-interaction). **No new variant CSS.**
- **Chevron:** 16px (12px xs/sm) inline SVG, `--tulpar-input-icon-default`; **rotates 180° on open**
  (`transform-origin:center`, overlay enter/exit duration). Hover lifts color to `text-secondary`.
- **Cursor:** `pointer` (it's a button, not text).
- **Clearable ✕:** 14px, shown only on hover/focus when a value exists; left of the chevron, separated by a
  faint 1px hairline (dropped at xs). Cross-fade 120ms. Clears + stops propagation.

### 5.2 Listbox surface (lighter sibling of popover)
- Portaled to top layer (§2.1); positioned by JS (fixed left/top). Radius **10px** (popover is 14px). Bg
  `--tulpar-overlay-surface-bg`, border `--tulpar-overlay-surface-border`, **one notch softer** overlay
  shadow + inset top-highlight hairline. Dark mode is fully token-driven.
- **Width** = trigger width by default (`min-width:<trigger>` set inline at open), may grow to content up
  to `max-width:min(92vw,28rem)`, never narrower than the trigger. **Gap** 6px to the trigger. **Flip** up
  when no room below (`data-side`).
- **Padding** 4px block / 0 inline (options own inline padding). **Max-height** reuses the popover cap with
  internal `overflow-y:auto; overscroll-behavior:contain` (~8–9 md options before scroll). Scrollbar reuses
  the popover's `currentColor`-keyed thin theme-aware treatment (auto-themes dark). Forced-colors reuses
  the popover block.

### 5.3 Option item — three independently legible signals
Geometry per size (min-height a touch shorter than the trigger; concentric 7px radius inside the 10px panel
with the 4px gutter):

| Size | Option min-h | Inline pad | Font | Icon↔label gap |
|---|---|---|---|---|
| xs | 26px | .5rem | .75rem | .375rem |
| sm | 30px | .625rem | .875rem | .5rem |
| md | 36px | .75rem | .875rem | .5rem |
| lg | 42px | .875rem | 1rem | .625rem |
| xl | 48px | 1rem | 1.125rem | .625rem |

Zones (logical, inline-start→end): leading icon (optional) · text block (label + optional muted
`description` second line, both single-line ellipsis) · fixed trailing check zone (always reserved, so
selection moving doesn't shift labels).

- **Default:** transparent bg, neutral text.
- **Active / hover (unified):** `--tulpar-color-bg-subtle` wash, **no rail, no check** — purely neutral, so
  it never reads as "selected." `transition: background 90ms` (instant feel).
- **Selected (committed value) — the signature, three stacked cues** (color never sole carrier):
  1. **2px brand left rail** (inset `box-shadow` / `::before` bar) — the one saturated element;
  2. **trailing checkmark** (16px / 14px xs-sm) in `--tulpar-color-brand-default`;
  3. faint **brand tint** `color-mix(in srgb, var(--tulpar-color-brand-default) 8%, surface)` (12% dark) —
     the established `selection.card.bgSelected` recipe; label weight bumps to ~550. Leading icon also
     adopts brand color.
- **Selected + active simultaneously:** keep rail + check + weight, **deepen** the fill
  (`color-mix(brand 12%, bg-subtle)`). Highlight never erases the selected rail/check.
- **Disabled option:** `--tulpar-color-text-muted` ~70%, `cursor:not-allowed`, no hover reaction, skipped by
  nav; icon at .6 opacity. A disabled *selected* option shows the check in muted (not brand).
- **Forced-colors:** active → `Highlight`/`HighlightText`; selected → keep check (`forced-color-adjust:auto`)
  + `outline:1px solid CanvasText` since tint/rail dissolve.

### 5.4 Group header & spacing
Non-interactive `role="presentation"` label: `.7rem`, weight 600, uppercase, `.06em` tracking,
`--tulpar-color-text-muted`, **sticky** within the scroll container (`position:sticky; top:0; background:
surface`) so group context stays visible while scrolling. First header gets reduced top padding.

### 5.5 Motion
Reuse overlay motion tokens, **snappier** than the popover. Open: opacity 0→1 + `scale(.98→1)` +
`translateY` 6px from the trigger-nearest edge (`transform-origin` per `data-side`), enter duration
(~160ms), overlay easing. Close: reverse, exit duration (~110ms); panel stays mounted through exit
(`data-exiting`). Chevron rotation synced. No per-option stagger. **`prefers-reduced-motion`:** opacity-only
enter/exit (reuse popover keyframes), chevron snaps.

### 5.6 Focus ring (touches the shipped input family — confirmed)
Add a soft 3px focus ring (`:focus-visible`) — `box-shadow: 0 0 0 3px var(--tulpar-color-focus-ring)` —
**to FormFieldBase** so the whole input family gains it (consistency). On invalid/warn the ring tints
danger/warn via `color-mix(border-invalid 32%, transparent)`. **Opt-out is token-based, not a new prop:**
override `--tulpar-color-focus-ring` (e.g. to `transparent`) at theme/scope/instance level — more flexible
than a boolean and zero new public API (YAGNI). Existing input tests/visuals are the regression gate.

### 5.7 Tokens
- **Reused as-is (no new tokens):** trigger → `--tulpar-input-*`; listbox surface/shadow/z/motion →
  `--tulpar-overlay-*`; option text/wash/border/link → `--tulpar-color-*`; selected accent →
  `--tulpar-color-brand-default`; focus ring → `--tulpar-color-focus-ring`.
- **Selected-option tint:** inline `color-mix(brand 8%/12%, surface)` (the existing selection-card recipe)
  — **no new semantic role** in v1. A dedicated `--tulpar-select-option-*` role is deferred until a second
  brand needs to retune it (internal-layering "promote on second consumer").
- Listbox padding override (4px vs popover's 16px) is inline; a dedicated `--tulpar-overlay-size-listbox-*`
  is deferred unless density needs independent tuning.

## 6. Wrappers (parity mandatory)

### 6.1 Angular (`@tulpar-ui/angular`) — signals only
`<tulpar-select-ng>` / `<tulpar-option-ng>` / `<tulpar-option-group-ng>`. Signal-based: `input()` for
props, `output()` for `change`, **`model()`** for two-way `value`. Form-association is native (no extra
CVA) → Reactive Forms works. No RxJS ([[feedback_angular_signals_only]]).
- **Slot projection gotcha:** slotted `<tulpar-option-ng>` children must project into the core
  `<tulpar-select>`'s shadow `<slot>`. The Angular wrapper wraps the core element, so verify the options
  reach the core slot and that state attrs (`data-active`/`data-selected`) mirror down to the rendered
  option ([[reference_angular_sidenav_wrapper_nesting]]). Verify in the Angular playground (rail/dark too).

### 6.2 Vue (`@tulpar-ui/vue`)
`<TulparSelect>` / `<TulparOption>` / `<TulparOptionGroup>`, SFC. `v-model` on `value`. **Slot forwarding
via the transparent carrier** (`<span style="display:contents" slot="label"><slot name="label"/></span>`)
— the documented Vue WC slot trap ([[feedback... vue slot carrier]]); applies to label/description/icon and
the option children. Apps configure `isCustomElement: tag => tag.startsWith('tulpar-')`.

## 9. Accessibility checklist (WCAG 2.2 AA)
- Accessible name from label, not placeholder; helper/error via `aria-describedby`.
- Full keyboard map (§4.2); focus visible (`:focus-visible` ring); focus restore on close.
- Meaning never by color alone (selected = check + rail + tint; invalid = border + icon + text).
- `prefers-reduced-motion` honored; `forced-colors` honored (trigger + options).
- Disabled options announced + skipped; empty/loading/error announced; `aria-busy` while loading.
- Touch targets: trigger min-height ≥ the size tier; rows comfortable (note: mobile-native fallback is
  deferred — desktop popover is used on touch; verify VoiceOver/TalkBack announce activedescendant).

## 10. RTL note (out of scope, non-blocking)
RTL is deferred, but to avoid a future rewrite: use **logical CSS properties** (`inline-start/-end`,
`margin-inline-*`) throughout the trigger/listbox/option styles, and keep the positioner inputs `dir`-aware.
The caret/check/clear-✕ sides and typeahead are direction-sensitive — logical properties make RTL a later
config flip, not a rewrite.

## 11. Out of scope (binding) + reserved surface
**Deferred (binding — do not pull forward):**
- **`options` data-prop** (array form) — slot is sufficient for v1. *Reserved:* the collection model is
  data-shaped, so an `options` prop can later feed `<tulpar-option>`s internally without breaking.
- **Combobox** (free-text filtering / in-popup search field) — Wave-2 next slice.
- **MultiSelect** — Wave-2 next slice. *Reserved:* `value` is single-string now; multi would add an array path.
- **Virtualization** (thousands of options) — incompatible with slot-owned nodes; later.
- **Mobile-native `<select>` fallback** — desktop popover on touch for v1. *Reserved:* keep the
  trigger/positioner API open to a native-on-mobile mode (e.g. a future `native-on-mobile` flag at the
  shell breakpoint).
- **`<tulpar-option-divider>`** as a standalone element — group spacing covers v1; reserved name if needed.
- `tulpar-open`/`tulpar-close` events — reserved names, not emitted in v1.
