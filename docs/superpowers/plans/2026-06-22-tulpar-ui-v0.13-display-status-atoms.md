# v0.13 Display & Status Atoms — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship seven small presentational atoms — Tag, Badge, Chip, Avatar, Skeleton, Spinner, Progress — as Lit Web Components with Angular/Vue wrappers, a shared promoted tone module, a new `atom` token block, Storybook stories, and ng+vue playground parity.

**Architecture:** Each atom is an independent Lit element under `packages/core/src/<name>/` that registers itself via a guarded `customElements.define`. The existing `toast/tone-resolver.ts` is promoted to `packages/core/src/_internal/tone/` and parameterized by CSS-var prefix so toast and the atoms share one engine; built-in tones stay attribute-only (semantic-token driven, zero inline styles — the grid hot path), `custom` is the inline-var escape hatch. Tones come from a new `atom` semantic token block (brand-bound, dark-flipped, contrast-tested). Wrappers forward every prop AND slot.

**Tech Stack:** Lit 3, TypeScript 5.4 (strict, `experimentalDecorators`), Vite library mode, Web Test Runner + @open-wc/testing (core), Vitest + happy-dom (wrappers, tokens), Angular 22 signals, Vue 3 SFC, Storybook 8 (`@storybook/web-components-vite`), Changesets.
