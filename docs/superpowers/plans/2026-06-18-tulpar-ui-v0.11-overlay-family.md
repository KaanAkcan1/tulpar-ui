# Tulpar UI v0.11 — Overlay Family Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Design phase:** This family has high visual/UX surface. Per the owner's standing instruction, invoke the `frontend-design` skill while writing component CSS / visual code (motion, surface, arrow, tone), and `ui-ux-pro-max` for any visual judgment not already pinned below. Concrete API/behavior values are pinned here; exact pixel/easing/shadow values for *surfaces* are resolved during implementation with frontend-design and then baked back into this plan's constants block.

**Goal:** Ship three new public Web Components — `<tulpar-tooltip>`, `<tulpar-toggletip>`, `<tulpar-popover>` — plus a package-private positioning primitive, new overlay tokens, Angular/Vue wrappers, Button integration, playground demos, and Storybook.

**Architecture:** A package-private positioner (`_internal/overlay/`) owns anchor resolution, placement compute, the `offset→flip→shift→arrow` JS pipeline (the guaranteed/reference path), optional native Popover-API top-layer + CSS-anchor-positioning enhancements, a single app-level portal root, and a stack manager. A shared `OverlayDelayController` owns cross-instance tooltip open/close delay + skip-delay. Three thin Lit elements compose the positioner and add their own ARIA contract, keyboard model, and animation. A constrained semantic `tone` prop (toggletip/popover only) maps 5 intents to brand families via new `--tulpar-color-overlay-*` tokens. Button's existing string `tooltip` attribute is refactored to delegate to `<tulpar-tooltip>`.

**Tech Stack:** Lit 3, TypeScript 5.4+ (strict, `experimentalDecorators`), `@web/test-runner` + `@open-wc/testing` (core), Vitest + happy-dom (wrappers), Storybook 8 (`@storybook/web-components-vite`), pnpm 9 catalogs, Vite library mode.

**Source of truth:** [`docs/superpowers/specs/2026-06-18-tulpar-ui-v0.11-overlay-family-design.md`](../specs/2026-06-18-tulpar-ui-v0.11-overlay-family-design.md). When this plan and the spec disagree, the spec wins for API/behavior; this plan wins for concrete values resolved during implementation.

> ⚠️ **COMPOSITION REWORKED (2026-06-19).** The slot-wrap model (`<tulpar-tooltip><button slot="trigger">`)
> was rejected — it inverts "tooltip on a button" into "button inside a tooltip." The composition is now
> **directive (framework) + `for`-id (core) + `…Ref` reference** — see spec §4 (authoritative). The engine
> (positioner, tokens, motion, a11y, contrast, element styles) was **salvaged** to this fresh branch; the
> trigger-binding, wrappers, and playgrounds are rebuilt. Any task below that says `slot="trigger"` or
> `tulpar-*-ng` wrapper components is **superseded** by the directive model. `tone` is **conventional hues**
> (info→gok, success→otuken, warning→ulgen, danger→al). See [[project_overlay_composition_decision]].

## Pinned decisions

1. **Convenience string prop = `text`** (tooltip/toggletip), on top of the overlay's child content (children win when present).
2. **`placement="auto"` is in-scope for v0.11.**
3. **Fallback positioning keeps the surface in the shadow root** (`position:fixed` + `--tulpar-overlay-z-index`); native Popover API promotes to the top layer. NO body portal (it strips shadow styles). Stack manager tracks open overlays regardless of DOM location.
4. **No `slot="trigger"`.** The trigger is a separate node, referenced by the overlay's `for="triggerId"` (core) or attached by a `tulparTooltip`/`…Ref` directive (frameworks). Rich content = the overlay element's children.
5. **Boolean prop naming is bare** (`arrow`, `flip`, `open`) — matches Button/Input.
6. **Directive naming keeps the `tulpar` prefix** (`tulparTooltip`/`tulparToggletip`/`tulparPopover` + `…Ref`) — bare `popover` collides with the native HTML attribute; bare `tooltip`/`v-tooltip` collide with ngx-bootstrap/floating-vue/Element Plus. Inline-vs-ref are separate inputs (no text/id ambiguity).

## Design constants (bake in; refine surface values with frontend-design during impl)

**Timing (motion tokens):**
- `overlay.delay.open` = `400ms` · `overlay.delay.close` = `120ms` · skip-delay grace window = `300ms`.
- `overlay.duration.enter` = `160ms` · `overlay.duration.exit` = `110ms` (exit < enter) · `overlay.easing` = `cubic-bezier(0.2,0.9,0.3,1)`.
- `prefers-reduced-motion: reduce` → transforms removed; enter/exit become opacity-only ≤`90ms`.

**Placement:** 12 placements `{top,bottom,left,right}-{start,center,end}` + `auto`. Default per element: tooltip/toggletip `top`, popover `bottom-start`. Resolved placement reflected as `data-placement` on the surface.

**Geometry defaults:** main-axis `offset` = `8` · `crossOffset` = `0` · `containerPadding` = `12` · `arrowBoundaryOffset` = `8` · `boundary` = `viewport` (also `scroll-parent` | an element — the element resolves `boundary` to the clipping rect it passes as the positioner's `viewport`) · arrow box ≈ `10px` (rotated 45° square). Tooltip `max-width` ≈ `32ch`; popover `max-height` = `min(viewport-space, --tulpar-overlay-popover-max-h)` with internal scroll.

**Events (all three elements):** `tulpar-open` · `tulpar-close` · `tulpar-toggle` (fires on every open/close transition with `{ open }` detail).

**Arrow defaults:** tooltip `arrow=true`, toggletip `arrow=true`, popover `arrow=false`. Arrow suppresses (opacity 0) when `shift` moves the surface so its center leaves the anchor span.

**`tone` → family map (conventional status hue, token-key spelling from `primitive/color.ts`):** `neutral→(default overlay surface)`, `info→gok` (blue), `success→otuken` (green), `warning→ulgen` (amber), `danger→al` (red — same family as input/selection invalid). Soft tonal surface, mirrored light/dark:

| role | light step | dark step |
|---|---|---|
| surface bg | `100` | `900` |
| on-surface (text) | `800` | `100` |
| border | `200` | `800` |
| arrow | = surface bg + border | — |

Focus ring inside a toned surface uses the neutral `--tulpar-color-overlay-focus-ring` token (NOT the tone family), verified ≥3:1 against the tinted bg. `tone` is a no-op under `forced-colors`.

**Verify all primitive ramp names** (`gok`, `ulgen`, `kizagan`, `erlik`, plus neutral families) against `packages/tokens/src/primitive/color.ts` before authoring; if a step is missing pick the nearest and note it.
