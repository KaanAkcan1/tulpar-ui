# Tulpar UI v0.8 — Sidebar Fixes + Rail Flyout Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix four sidebar bugs (group icon/label gap, theme-toggle label, Angular body padding, Angular group child-detection) and replace the rail's label-only tooltip with an enterprise flyout submenu.

**Architecture:** All behavior lives in the core Lit Web Components (`@tulpar-ui/shell`); the Angular/Vue wrappers only forward new props. New visual tokens are added to `@tulpar-ui/tokens` (brand layer) so component CSS stays semantic-token-only. The rail flyout reuses the existing `position: fixed` + `getBoundingClientRect()` machinery in `tulpar-nav-item.ts`, extending the single `.rail-flyout` element into a group panel when the item has children.

**Tech Stack:** Lit 3, TypeScript (strict, decorators), Web Test Runner + @open-wc/testing (core, real Shadow DOM), Vitest (tokens + wrapper smoke), Angular 22 signals, Vue 3 SFC, Changesets.

**Spec:** [`docs/superpowers/specs/2026-06-17-tulpar-ui-v0.8-sidebar-fixes-rail-flyout-design.md`](../specs/2026-06-17-tulpar-ui-v0.8-sidebar-fixes-rail-flyout-design.md)

### Task 3: A1 — Group item label flush against icon

A group item renders a `.chevron` with `margin-inline-start: auto`, competing with `.label { flex: 1 }`. Make `.label` the sole slack source so a group's label sits pixel-identical to a leaf's.

**Files:**
- Modify: `packages/shell/src/nav-item/tulpar-nav-item.styles.ts:99-102`
- Test: `packages/shell/src/nav-item/tulpar-nav-item.test.ts`

- [ ] **Step 1: Write the failing test**

This is a layout invariant. Because `.label{flex:1}` already eats slack, the *label's* left
offset can match even before the fix (false negative). The robust assertion is on the
**trailing edge**: with the competing chevron `margin-inline-start:auto`, a group's label
right edge stops short (chevron pushed by its own auto-margin AND the flexed label), leaving
a gap; after the fix the label fills to the chevron with the same trailing gap as a leaf's
trailing chip. Assert both label-left parity AND that the group label's width is not
collapsed. Fix the leaf/group to identical widths by giving both a trailing element. Add to
`tulpar-nav-item.test.ts`:

```ts
it("group item label aligns and fills identically to a leaf item with a trailing chip", async () => {
  const leaf = await fixture<TulparNavItem>(html`
    <tulpar-nav-item href="/a" label="Alpha" icon="<svg width='18' height='18'></svg>" count="3"></tulpar-nav-item>
  `);
  const group = await fixture<TulparNavItem>(html`
    <tulpar-nav-item label="Group" icon="<svg width='18' height='18'></svg>">
      <tulpar-nav-item href="/c" label="Child"></tulpar-nav-item>
    </tulpar-nav-item>
  `);
  await leaf.updateComplete;
  await group.updateComplete;
  const leafLabel = leaf.shadowRoot!.querySelector(".label") as HTMLElement;
  const groupLabel = group.shadowRoot!.querySelector(".label") as HTMLElement;
  // Same left offset (icon + gap), AND the group label is not gap-collapsed:
  expect(Math.abs(groupLabel.getBoundingClientRect().left - leafLabel.getBoundingClientRect().left))
    .to.be.lessThan(1);
  // The group chevron must sit flush at the trailing edge — assert the label fills to it.
  const groupRow = group.shadowRoot!.querySelector("a, button") as HTMLElement;
  const chevron = group.shadowRoot!.querySelector(".chevron") as HTMLElement;
  const rowRight = groupRow.getBoundingClientRect().right;
  // chevron right edge should be within the row's right padding (~0.75rem ≈ 12px), i.e.
  // the chevron is NOT pushed away from the trailing edge by a competing auto-margin.
  expect(rowRight - chevron.getBoundingClientRect().right).to.be.lessThan(16);
});
```

## Chunk 2: A2 — Theme toggle target-mode label

The utility theme button always reads "Theme". It must show the **target** mode ("Dark" in light, "Light" in dark), overridable via two new props.

**Design note (revised from spec's `data-dark` shell-reflection):** reflecting `data-dark`
from the shell onto the *slotted* sidenav breaks under the Angular wrapper — the slotted
element is `<tulpar-sidenav-ng>` (`display:contents`), so `data-dark` would land on the
wrapper, not the inner core `<tulpar-sidenav>` that reads it (the same wrapper-nesting
class as the A4 bug). Instead, the **sidenav observes the global `.dark` class on
`document.documentElement`** (which `tulpar-shell` already toggles as the source of truth)
via a `MutationObserver`, derives an internal `_dark` state, and reflects `data-dark` on
**itself** so `:host([data-dark])` CSS and the label text share one signal. This is
wrapper-agnostic (works identically in core/Angular/Vue) and keeps icon+text in sync.

- [ ] **Step 3: Implement**

```ts
@state() private _dark = false;
private _darkObserver: MutationObserver | null = null;

private _syncDark = () => {
  const dark = document.documentElement.classList.contains("dark");
  if (dark !== this._dark) {
    this._dark = dark;
    this.toggleAttribute("data-dark", dark);
  }
};
```
In `connectedCallback()`, after the existing setup:

```ts
this._syncDark();
this._darkObserver = new MutationObserver(this._syncDark);
this._darkObserver.observe(document.documentElement, {
  attributes: true,
  attributeFilter: ["class"],
});
```
In `disconnectedCallback()`:

```ts
this._darkObserver?.disconnect();
this._darkObserver = null;
```

> If `shadow` were nested inside `ShellTokens.shell`, it would wrongly emit
> `--tulpar-shell-shadow-flyout`. Do NOT nest them under `shell`.

Emitting `--tulpar-shadow-sm`/`-md` also resolves the latent dangling references in
`tulpar-nav-item.styles.ts` (`--tulpar-shadow-sm`) and `tulpar-button.styles.ts`
