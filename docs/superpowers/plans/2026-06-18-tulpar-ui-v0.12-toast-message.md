# Tulpar UI v0.12 — Toast + Message Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development (fresh subagent per task, two-stage review). Steps use checkbox (`- [ ]`) syntax. After approval, save the canonical copy of this plan to `docs/superpowers/plans/2026-06-18-tulpar-ui-v0.12-toast-message.md` (Task 0) and execute from there.

**Goal:** Ship a transient-feedback family — an imperative `toast` + `message` service driven by a single auto-mounted `<tulpar-toaster>` engine, plus a `<tulpar-toast>` element, tokens, and Angular/Vue wrappers + playground demos.

**Architecture:** Framework-agnostic engine in `@tulpar-ui/core/_internal/toaster` (queue, per-location stacking, timer controller, portal/theme root, a11y region). A thin `toast-service` exposes the imperative API. `<tulpar-toast>` is pure presentation/interaction. Wrappers expose the service idiomatically (Angular DI/signals, Vue composable) and forward the element's slots.

**Tech Stack:** Lit 3 + TS (decorators, `useDefineForClassFields:false`), `@tulpar-ui/tokens` (TS→CSS), Web Test Runner + `@open-wc/testing` (core), Vitest + happy-dom (wrappers), Angular v22 signals (no RxJS), Vue 3 SFC.

**Spec (authoritative):** `docs/superpowers/specs/2026-06-18-tulpar-ui-v0.12-toast-message-design.md`

---

## Context

v0.1–v0.11 shipped Button, inputs, selection, shell, and the overlay family (anchored tooltip/toggletip/popover). There is **no app-level transient feedback** primitive. This adds Toast (corner, rich, stacked) + Message (top-center, single-line, grouped) as a single imperative service — the pattern every leading system (Sonner/Ant/Element/React-Aria/Carbon) converges on. Dialog (modal) is deliberately a **separate later spec** and out of scope here (spec §13).

Established patterns to **reuse, not reinvent**:
- **Portal + theme mirroring + Esc stack:** `packages/core/src/_internal/overlay/overlay-root.ts` (`getOverlayRoot`, `syncTheme` MutationObserver on `documentElement` for `dark`/`data-brand`, `pushOverlay`/`popOverlay`, `__resetOverlayRootForTest`). The toaster region root mirrors this exactly.
- **Token interface shape:** `packages/tokens/src/semantic/overlay.ts` (`OverlayTokens`/`OverlayToneTokens`/`OverlayMotionTokens`) — model `FeedbackTokens` on it. Contrast test pattern: `packages/tokens/src/semantic/overlay.contrast.test.ts`.
- **Component layout:** each family is a folder of `index.ts` + `tulpar-X.ts` + `tulpar-X.styles.ts` + `tulpar-X.test.ts` (see `packages/core/src/switch/`). Styles use semantic tokens only + `var()` fallbacks + `:focus-visible` + forced-colors `CanvasText` + reduced-motion (see `tulpar-switch.styles.ts`).
- **`warn-dev`** helper: `packages/core/src/_internal/warn-dev.ts` for dev-mode warnings.
- **Angular wrapper:** `display:contents` host, `CUSTOM_ELEMENTS_SCHEMA`, `[attr.*]` bindings, `NgComponentOutlet` for icon-component slots, imperative attr application where DomSanitizer blocks bindings (`packages/angular/src/switch/tulpar-switch.component.ts`). **No RxJS.**
- **Vue wrapper:** SFC forwarding named slots through `<span style="display:contents" slot="…">` carriers (CLAUDE.md gotcha; `packages/vue/src/switch/TulparSwitch.vue`).

Constraints: only **semantic** tokens in component CSS (ESLint-enforced on `*.styles.ts`); `_internal` never re-exported/cross-imported; never reference primitive `--tulpar-primitive-*` in styles; no AI attribution in commits ([[feedback_no_ai_attribution_commits]]).

## File Structure

**core** (`packages/core/src/`)
- `_internal/toaster/queue.ts` — per-location queue + `maxVisible` overflow + grouping (Message). Pure logic, no DOM.
- `_internal/toaster/timer-controller.ts` — per-toast countdown with pause/resume (hover/focus), `onAutoClose`.
- `_internal/toaster/toaster-root.ts` — singleton portal root + theme mirror + a11y region landmark + F6 hotkey (modeled on `overlay-root.ts`).
- `_internal/toaster/stacking.ts` — Sonner collapse math (scale/lift) + `expand` layout, applied as inline transforms.
- `toast/tulpar-toast.styles.ts` — tonal surfaces, perimeter timer ring, anatomy, swipe transforms, dark/forced-colors/reduced-motion.
- `toast/tulpar-toast.ts` — `<tulpar-toast>` element (presentation + swipe + close + slots `title`/`description`/`icon`/default).
- `toast/tone-resolver.ts` — built-in tone → token mapping; `custom` family-name/hex/override resolution.
- `toast/toast-service.ts` — imperative `toast` + `message` objects (sugar, `promise`, `update`, `dismiss`).
- `toast/index.ts` — public exports; add `export * from "./toast";` to `src/index.ts`.

**tokens** (`packages/tokens/src/`)
- `semantic/feedback.ts` — `FeedbackTokens` interface + key list.
- `semantic/feedback.contrast.test.ts` — the tone×mode contrast matrix.
- brand bindings in `brand/tulpar/light.ts` + `dark.ts`; register in `semantic/types.ts` + the brand index/generator.

**angular** (`packages/angular/src/`)
- `toast/tulpar-toast.providers.ts` — `provideTulparToast()`, `TulparToast`/`TulparMessage` injectables wrapping the core service.
- `toast/tulpar-toast.component.ts` — optional `<tulpar-toast-ng>` for declarative/rich slot path.
- export from `public-api.ts`.

**vue** (`packages/vue/src/`)
- `toast/index.ts` — `TulparToastPlugin`, `useTulparToast()`, `useTulparMessage()`.
- `toast/TulparToast.vue` — declarative/rich SFC with slot carriers.
- export from `src/index.ts`.

**playgrounds**
- `apps/playground-ng/src/app/demos/toast-demo.component.ts` + `message-demo.component.ts` + routes + sidenav.
- `apps/playground-vue/src/components/ToastDemo.vue` + `MessageDemo.vue` + router + sidenav.

**docs**: Storybook stories under `apps/docs` (match existing story locations).

## Risks / watch-items
- **Lit duplicate-instance crash** ([[reference_lit_directive_dual_instance_crash]]): use plain style strings in the hot stacking/timer paths, not `styleMap`/`unsafeSVG`; dedupe lit in playground Vite.
- **Live region inside modal** (spec §5.3): keep the toaster region at document top level.
- **WTR gotchas** ([[reference_wtr_shell_test_gotchas]]): pin mobile-breakpoint viewport in layout-dependent tests; kill zombie chrome on the port.
- **Angular DomSanitizer** blocks some `[attr.*]`: apply such attrs imperatively via `afterRenderEffect` (see switch wrapper).
