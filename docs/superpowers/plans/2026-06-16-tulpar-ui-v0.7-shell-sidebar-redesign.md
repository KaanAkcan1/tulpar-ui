# Tulpar UI v0.7 — Shell Sidebar Redesign Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Elevate the `@tulpar-ui/shell` sidebar to enterprise/product-ready quality (lit-pill active state, sections, rich nav-items, utility row, position left/right, rail overflow fix), wire the official brand assets through the playgrounds, and raise the Storybook/playground polish bar — all without changing any existing token color value.

**Architecture:** Phase **v0.7a** touches the design-system packages (`@tulpar-ui/tokens`, `@tulpar-ui/shell`, Angular/Vue wrappers): re-bind a few shell semantic tokens to existing primitives, add new shell tokens (each resolving to an existing primitive or its rgba), redesign `tulpar-nav-item`, add a new `tulpar-nav-section` element, and expand `tulpar-sidenav`. Phase **v0.7b** is consumer polish (brand assets, playground menu IA + pages + footers, Storybook fixes/deepening) and depends on v0.7a being built. Core components are tested in a real browser via `@web/test-runner` + `@open-wc/testing`; wrappers get smoke tests; token bindings are snapshot-tested.

**Tech Stack:** Lit 3, TypeScript 5.4 (strict, `experimentalDecorators`), pnpm 9 workspaces, Vite library mode, `@web/test-runner`, Vitest (wrappers), Angular 22 (signals), Vue 3.4, Storybook 8.

**Spec:** [`docs/superpowers/specs/2026-06-16-tulpar-ui-v0.7-shell-sidebar-redesign-design.md`](../specs/2026-06-16-tulpar-ui-v0.7-shell-sidebar-redesign-design.md)

**Binding constraint (read first):** NEVER edit a primitive color value (`packages/tokens/src/primitive/color.ts`). You may *re-bind* a shell semantic token to a different existing primitive, and you may *add* new shell tokens that resolve to an existing primitive or an `rgba()` of one. Component CSS uses `var(--tulpar-shell-…, <fallback>)` only (no primitive tokens — enforced by ESLint on `*.styles.ts`); the fallback must mirror the resolved primitive hex. Zero invented hex.

**Working branch:** `feat/brand-assets-integration` (do NOT use a worktree — owner preference). Commit frequently to this branch.

import { LitElement, html, nothing } from "lit";
import { property } from "lit/decorators.js";
import { navSectionStyles } from "./tulpar-nav-section.styles";

let _uid = 0;

export class TulparNavSection extends LitElement {
  static override styles = navSectionStyles;

  @property({ type: String }) label?: string;

  private readonly _labelId = `tulpar-nav-section-${++_uid}`;

  override render() {

      ${this.label
        ? html`<div class="section-label" id=${this._labelId}>${this.label}</div>`

        role="group"
        aria-labelledby=${this.label ? this._labelId : nothing}
        class="section-items"
      >

      </div>
    `;
  }
}

if (!customElements.get("tulpar-nav-section")) {
  customElements.define("tulpar-nav-section", TulparNavSection);
}
