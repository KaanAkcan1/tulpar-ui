# Tulpar UI v0.10 — Selection Family Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship 5 new public Web Components — `<tulpar-switch>`, `<tulpar-checkbox>`, `<tulpar-radio>`, `<tulpar-radio-group>`, `<tulpar-checkbox-group>` — plus Angular/Vue wrappers and Storybook, sharing two unexported abstract bases.

**Architecture:** Two unexported abstract Lit bases mirror the `FormFieldBase` pattern without reusing it: `SelectionControlBase` (form-associated; switch/checkbox/radio) and `SelectionGroupBase` (orchestrates slotted children; radio-group/checkbox-group). The ~50-line message-row/status renderer is copied from `FormFieldBase` per the `_internal` ≤50-line copy rule. All styling uses new `--tulpar-selection-*` / `--tulpar-switch-*` semantic tokens with literal `var()` fallbacks (primitive-token lint stays satisfied). Visual personality is **tactile/physical**: springy overshoot motion, layered "physical disk" switch thumb, tinted+lifted selectable cards.

**Tech Stack:** Lit 3, TypeScript 5.4+ (strict, `experimentalDecorators`), `@web/test-runner` + `@open-wc/testing` (core), Vitest + happy-dom (wrappers), Storybook 8 (`@storybook/web-components-vite`), pnpm 9 catalogs, Vite library mode.

**Source of truth:** [`docs/superpowers/specs/2026-06-17-tulpar-ui-v0.10-selection-family-design.md`](../specs/2026-06-17-tulpar-ui-v0.10-selection-family-design.md). When this plan and the spec disagree, the spec wins for API/behavior; this plan wins for concrete visual values (resolved from the 2026-06-17 design subagent reports + user fork choices).

## Resolved design constants (bake these in — do not re-derive)

**Personality (user-chosen forks):** springy motion · layered physical thumb shadow · tinted+lift cards.

**Per-tier scale** (px; label font in rem-equivalent shown as px; control box = checkbox edge & radio outer; switch derives from it):

| tier | control box | gap | label fs | desc fs | touch pad | card pad | card radius |
|---|---|---|---|---|---|---|---|
| xs | 14 | 6 | 12 | 11 | 8 | 8 | 7 |
| sm | 16 | 8 | 13 | 12 | 6 | 10 | 8 |
| md | 18 | 10 | 14 | 12 | 5 | 12 | 9 |
| lg | 20 | 12 | 16 | 13 | 4 | 14 | 10 |
| xl | 22 | 14 | 18 | 14 | 3 | 16 | 11 |

Switch geometry derived per tier: `track-h = box + 4`, `track-w = round(track-h * 1.75)`, `thumb = track-h - 4`, `travel = track-w - thumb - 4`.
→ md: track 22×38, thumb 18, travel 16. (xs 18×31/14/9 · sm 20×35/16/11 · lg 24×42/20/14 · xl 26×46/22/16.)

**Motion (springy):**
- Switch thumb slide: `transform: translateX(...)` `240ms cubic-bezier(0.34,1.3,0.64,1)`.
- Switch track color: `200ms cubic-bezier(0.2,0,0,1)`.
- Thumb icon crossfade: `160ms ease`. Thumb press squish (`width +3px`): `120ms cubic-bezier(0.2,0,0,1)`.
- Checkbox box fill (bg+border): `140ms cubic-bezier(0.2,0,0,1)`. Check draw (`stroke-dashoffset 1→0`): `200ms cubic-bezier(0.65,0,0.35,1) 40ms`.
- Indeterminate dash: `transform: scaleX(0.4→1)` `160ms cubic-bezier(0.2,0,0,1)` + fade.
- Radio dot pop: `width/height 0→40% 180ms cubic-bezier(0.34,1.4,0.64,1)`.
- Card hover: border+bg `140ms`, shadow `160ms`, `transform: translateY(-1px) 120ms` (all `cubic-bezier(0.2,0,0,1)`); card press `translateY(0) scale(.995)`.
- Control press: box `scale(.94)` (radio) / `.96` (checkbox) `90ms`.
- Loading spin: `700ms linear infinite`. Reduced-motion → spinner becomes opacity pulse `tulpar-sel-pulse 1.2s ease-in-out infinite`, all transitions `none`, check shown fully drawn.

**Focus ring:** `outline: 2px solid var(--tulpar-selection-focus-ring, …); outline-offset: 2px` (switch track 3px) on `:focus-visible`; card ring via `.root:has(:focus-visible)`; `@media (forced-colors: active){ outline-color: CanvasText }`.

**Switch thumb shadow (layered):** light `0 1px 2px rgba(11,8,4,.28), 0 2px 6px -1px rgba(11,8,4,.18), inset 0 1px 0 rgba(255,255,255,.7)`; dark `0 1px 2px rgba(0,0,0,.45), inset 0 1px 0 rgba(255,255,255,.4)`.

**Card selected:** `border-color: var(--_sel-fill)`, `background: var(--tulpar-selection-card-bg-selected, color-mix(in oklch, var(--_sel-fill) 8%, surface))` (dark 12%), `box-shadow: 0 0 0 1px var(--_sel-fill) inset`. Suppressed under `invalid`/`disabled`.
