# Migrating from Tulpar UI v0.4.x to v0.5

v0.5 is a purely additive release. There are **no breaking changes** to
`<tulpar-button>` or `<tulpar-button-group>`. Updating to v0.5 requires no
migration work if you only use the button family; everything below is new
functionality you can adopt at your own pace.

See the full design spec: [`docs/superpowers/specs/2026-06-09-tulpar-ui-v0.5-input-family-design.md`](../superpowers/specs/2026-06-09-tulpar-ui-v0.5-input-family-design.md)

---

## What's new — three public custom elements

### `<tulpar-text-input>`

Single-line text input. Supported types: `text | email | url | tel | search | password`.

- `type="password"` ships a built-in reveal toggle (opt out with `no-reveal-toggle`)
- `type="search"` auto-enriches with a leading search icon and enables `clearable` by default
- Integrated mask engine (see [Mask engine](#mask-engine) below)
- Optional character counter (`show-count`)
- Clipboard affordances: `copyable` / `pastable` (with visual confirmation)

### `<tulpar-textarea>`

Multi-line text input.

- `autosize` on by default — grows from `min-rows` (default 2) to `max-rows` (default 6)
- Manual resize direction via `resize="none|both|horizontal|vertical"` (default `vertical`)
- Setting explicit `rows` disables autosize
- Optional overlay character counter (`show-count`)
- Clipboard affordances: `copyable` / `pastable`
- No prefix/suffix slots (visual incompatibility with multi-line layout)

### `<tulpar-number-input>`

Numeric entry with locale-aware formatting.

- `value` is `number | null`; `null` represents empty (controlled by `allow-empty`, default `true`)
- On focus, the formatted display is stripped to raw digits for editing; on blur the value is
  re-formatted and clamped to `[min, max]`
- Stepper buttons with long-press acceleration (`step-hold-delay` / `step-hold-interval`)
- Keyboard: `↑` / `↓` step by `step`; `Shift+↑↓` × 10; `Ctrl/⌘+↑↓` × 100; `Home` / `End` jump to `min` / `max`
- Integer-only mode: `integer-only` — rejects decimal separators with a shake animation
- `format-style="percent"` follows Intl semantics: value `0.125` renders as `12.5%`

## Wrappers

| Framework | TextInput | Textarea | NumberInput |
|---|---|---|---|
| Angular | `<tulpar-text-input-ng>` | `<tulpar-textarea-ng>` | `<tulpar-number-input-ng>` |
| Vue | `<TulparTextInput>` | `<TulparTextarea>` | `<TulparNumberInput>` |

Angular wrappers are signal-based (`input()` / `output()`). Vue wrappers support `v-model`.

## Shared anatomy

All three components share the following anatomy, implemented in the (unexported)
internal `FormFieldBase`.

### Label

Provide via attribute or slot — slot wins when both are present:

```html
<!-- attribute -->
<tulpar-text-input label="Email address"></tulpar-text-input>

<!-- slot wins over attribute -->
<tulpar-text-input label="fallback">
  <span slot="label">Email <em>address</em></span>
</tulpar-text-input>
```

### Label positions

`label-position` accepts five values:

| Value | Behaviour |
|---|---|
| `top` | Label rendered above the field (default when label is present) |
| `float` | PrimeNG-style "over" — label rests inside the input when empty/unfocused, animates fully **above the top border** on focus or when the field has a value. The field permanently reserves the above-border space, so there is zero layout shift. |
| `float-in` | Label rests inside (same resting position as `float`), floats to the **inside-top-edge** on focus/value |
| `float-on` | Label sits statically on top of the border in a notch — the only mode that cuts the border line. |
| `none` | No label element rendered; `label` attribute is wired to `aria-label` for accessibility |

> At rest (empty, unfocused), `float` and `float-in` are visually identical.
> They differ only in where the label travels on focus/value.

### Message row

A single always-reserved row below the field displays helper, error, or warning text.
Only one message is shown at a time (priority: `error-text` > `warn-text` > `helper-text`).

```html
<tulpar-text-input
  helper-text="We'll never share your email"
  error-text="Enter a valid email address"
  warn-text="Looks unusual — is this correct?"
  invalid
></tulpar-text-input>
```

Opt out of the reserved space (for dense layouts) with `no-message-space`.

### Status taxonomy

Three orthogonal status attributes:

| Attribute | Meaning |
|---|---|
| `invalid` | Validation failed — shows `error-text`, red border, error icon |
| `warn` | Soft warning — shows `warn-text`, amber border, warning icon |
| `validating` | Async check in progress — shows spinner icon, sets `aria-busy` |

`validating` is orthogonal and coexists with `invalid` or `warn`. A field can be
simultaneously `validating invalid` (re-validating an already-known-bad value).

### Necessity indicator

`required` + `necessity-indicator` control what appears next to the label:

| `necessity-indicator` | Required | Optional |
|---|---|---|
| `icon` (default) | `*` marker | nothing |
| `label` | `(required)` text | `(optional)` text |
| `none` | nothing | nothing |

### Size

`size` accepts `xs | sm | md | lg | xl` (default `md`).

At `xs`, action buttons (clear, copy, paste, reveal toggle, steppers) are
automatically hidden — they cannot meet the 44 pt touch-target minimum at
that density.

### Variant

`variant` accepts `outlined | filled | underlined | ghost` (default `outlined`).

`ghost` + `size="xs"` + `label-position="none"` + `no-message-space` is the
recommended recipe for inline grid-cell or table editors.

### Prefix / suffix slots

```html
<tulpar-text-input>
  <span slot="prefix">@</span>
  <span slot="suffix">.com</span>
</tulpar-text-input>
```

By default, prefix/suffix content is purely decorative (pointer-events disabled).
Set `prefix-interactive` or `suffix-interactive` to allow clicks (e.g., a clickable
icon button in the suffix).

### Clipboard affordances

`copyable` adds a copy button that writes the current value to the clipboard.
`pastable` adds a paste button that reads from the clipboard into the field.
Both show a brief visual confirmation (checkmark) after the action.

## Mask engine

`<tulpar-text-input>` accepts a `mask` attribute for structured input formatting.

### Token alphabet

| Token | Accepts | Notes |
|---|---|---|
| `9` | Digit (0–9) | Unless escaped with `\` |
| `A` | Uppercase letter | Forces input to uppercase |
| `a` | Lowercase letter | Forces input to lowercase |
| `L` | Any letter | Preserves case |
| `S` | Special / punctuation character | |
| `*` | Any printable character | |
| `\X` | Literal `X` | Escapes any token character |
| Any other character | Literal — auto-inserted | |

### Backslash escaping — critical for phone masks

The digit `9` is always a token unless preceded by `\`. Turkish phone numbers
start with literal `+90`, where `9` must be escaped:

**HTML attribute / Vue template (single backslash):**
```html
<tulpar-text-input mask="+\90 (999) 999 99 99"></tulpar-text-input>
```

**JS template literal / Angular backtick template (double backslash, because `\9`
is not a recognized JS escape sequence and the string must contain a literal `\`):**
```ts
// Angular component template (inside backtick string)
`<tulpar-text-input-ng mask="+\\90 (999) 999 99 99"></tulpar-text-input-ng>`

// Or set it programmatically:
el.mask = "+\\90 (999) 999 99 99";
```

> Rule of thumb: the actual string stored in the DOM attribute must be
> `+\90 (999) 999 99 99` — one backslash. In JS string literals, one DOM
> backslash requires two source backslashes (`\\`).

### Mask attributes

| Attribute | Type | Default | Purpose |
|---|---|---|---|
| `mask` | string | — | Mask pattern |
| `mask-emit` | `masked \| raw` | `masked` | What the `value` / `change` event exposes |
| `mask-display` | `eager \| lazy` | `eager` | Eager shows slot-chars immediately; lazy only on focus/type |
| `mask-slot-char` | string | `_` | Placeholder character for unfilled slots |

The `.rawValue` property is always available regardless of `mask-emit`.

## NumberInput — hybrid format API

Shorthand attributes cover the common cases:

| Attribute | Type | Example |
|---|---|---|
| `format-style` | `decimal \| currency \| percent` | `format-style="currency"` |
| `currency` | string (ISO 4217) | `currency="TRY"` |
| `locale` | string (BCP 47) | `locale="tr-TR"` |
| `min-fraction-digits` | number | `min-fraction-digits="2"` |
| `max-fraction-digits` | number | `max-fraction-digits="2"` |
| `use-grouping` | boolean | `use-grouping` (default on) |
| `format-prefix` | string | `format-prefix="~"` |
| `format-suffix` | string | `format-suffix=" kg"` |

For advanced cases (compact notation, accounting style, `roundingMode`, etc.),
set the JS-only `.formatOptions` property with a full `Intl.NumberFormatOptions`
object. `formatOptions` overrides the shorthand attributes entirely.

```ts
const el = document.querySelector("tulpar-number-input");
el.formatOptions = { notation: "compact", compactDisplay: "short" };
```

In Vue, the wrapper handles this automatically — pass `:format-options="{ ... }"`.

## Quick-start examples

### Sign-in form (Web Components)

```html
<form>
  <tulpar-text-input
    name="email"
    type="email"
    label="Email address"
    autocomplete="username"
    required
  ></tulpar-text-input>

  <tulpar-text-input
    name="password"
    type="password"
    label="Password"
    autocomplete="current-password"
    required
  ></tulpar-text-input>

  <tulpar-button type="submit">Sign in</tulpar-button>
</form>
```

### Masked Turkish phone (HTML form, single backslash in attribute)

```html
<form>
  <tulpar-text-input
    name="phone"
    type="tel"
    label="Phone number"
    mask="+\90 (999) 999 99 99"
    mask-emit="raw"
    autocomplete="tel"
  ></tulpar-text-input>
</form>
```

`mask-emit="raw"` means `FormData` receives only the digits (e.g., `9053212345678`),
not the formatted display string.

### Currency entry (Vue v-model)

```vue
<script setup lang="ts">
import { ref } from "vue";
import { TulparNumberInput } from "@tulpar-ui/vue";

const price = ref<number | null>(null);
</script>

<template>
  <TulparNumberInput
    v-model="price"
    label="Unit price"
    format-style="currency"
    currency="TRY"
    locale="tr-TR"
    :min-fraction-digits="2"
    :max-fraction-digits="2"
    :min="0"
    helper-text="VAT excluded"
  />
</template>
```

### Grid-cell / table editor recipe

Use `ghost` variant with `xs` size, no label, and no message row for the
most compact possible inline editor:

```html
<tulpar-number-input
  variant="ghost"
  size="xs"
  label-position="none"
  no-message-space
  :value="cellValue"
></tulpar-number-input>
```

## Deferred / out of scope

The following were considered for v0.5 and explicitly deferred:

- **Dedicated `PasswordInput`** — strength meter, breach-check integration
- **`PhoneInput` component** — international formatter (libphonenumber / intl-tel-input)
- **`SearchInput` component** — debounced async search, suggestions dropdown
- **Optional mask tokens** — e.g., `?` prefix to mark a slot as skippable
- **Big-int mode** for NumberInput — values beyond `Number.MAX_SAFE_INTEGER`
- **`<tulpar-form-field>` wrapper** — standalone label/message row for custom controls
- **React wrapper**
